# macOS DMG installer for `primer-gui`

**Status:** Design approved 2026-05-13. Implementation plan pending.

## Goal

Produce a signed and notarized macOS `.dmg` installer for the Tauri-based desktop GUI (`primer-gui`) so Horst can hand it to evaluators who can install it with no Gatekeeper friction. Private distribution only — App Store submission is explicitly out of scope.

A fresh install must launch into a working session-picker, allow backend configuration in Settings, and start a Socratic conversation against a populated knowledge base **without any further setup** beyond entering an API key (or pointing at a local Ollama instance).

## Non-goals

- Universal binary (Intel + Apple Silicon). Apple Silicon only.
- Mac App Store submission. Developer ID distribution only.
- Speech mode (`--speech` feature). Brings espeak-ng system dependency, mic permission entitlement, and ~50 MB of voice model files; major scope expansion.
- Hybrid retrieval (`embedding` feature). Triggers a ~570 MB BGE-M3 model download from `cdn.pyke.io` on first launch; not acceptable for an offline-friendly evaluation bundle.
- GitHub Actions release workflow that produces the DMG. Manual build via `scripts/build-dmg.sh` first; CI is a follow-up.
- Auto-update channel.

## Distribution and signing

Signing identity: `Developer ID Application: Horst Herb (X5DWXB4283)` — present in the login keychain and visible to `security find-identity -p codesigning -v`.

Notarization: Apple notary via `xcrun notarytool`, driven by Tauri 2's bundler. Credential is an App Store Connect API key (`.p8` file). Three env vars feed Tauri:

- `APPLE_API_ISSUER` — Issuer ID
- `APPLE_API_KEY` — Key ID
- `APPLE_API_KEY_PATH` — absolute path to the `.p8` file

The notary stamp is stapled to the `.app` inside the DMG, so Gatekeeper accepts it offline on the evaluator's machine.

Bundle identifier: `org.theprimer.gui` — unchanged from the existing `tauri.conf.json`. Does not conflict with Horst's existing App Store apps (`com.hherb.*`, `com.bmlibrarian.*`). Developer ID signing does not require a provisioning profile, so no App ID registration is needed at developer.apple.com for this identifier.

Minimum macOS version: 11.0 (Big Sur). The Apple Silicon line starts at 10.16 anyway; 11.0 is the conservative floor that covers every Mac that can run an `aarch64-apple-darwin` binary.

## Architecture

Seven touch points in the repo. All changes localized to the GUI crate plus a build script, README, and `.gitignore`.

### 1. `src/crates/primer-gui/tauri.conf.json`

Flip the `bundle` block from off to on, narrow to macOS targets, and configure signing.

```jsonc
{
  // ... existing app config unchanged ...
  "bundle": {
    "active": true,
    "targets": ["dmg", "app"],
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ],
    "resources": ["../../../../data/seed/*.jsonl"],
    "copyright": "© 2026 Horst Herb",
    "category": "public.app-category.education",
    "shortDescription": "A Socratic AI learning companion for children",
    "longDescription": "The Primer is an offline-friendly Socratic learning companion. It asks more questions than it answers, catches parroting, suggests breaks, and remembers what was discussed across sessions.",
    "macOS": {
      "minimumSystemVersion": "11.0",
      "signingIdentity": "Developer ID Application: Horst Herb (X5DWXB4283)",
      "entitlements": "entitlements.plist"
    }
  }
}
```

The icon array lists files relative to `tauri.conf.json` (i.e. `src/crates/primer-gui/icons/...`). The full set is generated by `cargo tauri icon` from a single 1024×1024 source PNG.

The resources glob climbs four levels from `src/crates/primer-gui/` to the repo root and grabs every `*.jsonl` in `data/seed/`. Tauri 2's bundler copies these into `Primer.app/Contents/Resources/_up_/_up_/_up_/_up_/data/seed/*.jsonl` by default (it preserves the relative path structure with `_up_` segments standing in for `..`). The runtime path resolver in step 3 must account for this.

**Fallback if upward globs are rejected**: add a `build.rs` step in `primer-gui` that copies `../../../../data/seed/*.jsonl` into `src/crates/primer-gui/resources/seed/` at build time, and reference `resources: ["resources/seed/*.jsonl"]` instead. Cleaner bundle layout (`Resources/resources/seed/`), at the cost of a build-script copy. Decided at implementation time based on whether Tauri 2 accepts the direct upward glob.

### 2. `src/crates/primer-gui/entitlements.plist` (new file)

Minimal entitlements for a Developer-ID-signed Tauri/wry application:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>
</dict>
</plist>
```

The two entitlements above are required for WebKit's JIT to function inside a hardened (signed + notarized) binary; without them the WebView crashes on launch with a code-signing error. `com.apple.security.cs.disable-library-validation` is **not** included in the initial design — if notarization or runtime rejects a bundled dylib, it is added in a follow-up iteration rather than enabled preemptively (broad attack surface, easy to add later, hard to remove once shipped).

App Sandbox is **not** enabled. Sandboxing is an App Store requirement; for Developer ID distribution it is optional and adds friction (entitlement plumbing for filesystem access, mic, etc.). The Primer reads/writes `~/.primer/<slug>.db` and the bundled `Resources/` dir — both work without sandboxing.

### 3. `src/crates/primer-gui/src/main.rs` — packaged-app seed path resolution

At startup, before the engine constructs the knowledge base, detect whether the binary is running inside a `.app` bundle. If so, resolve the bundled seed-dir path and set `PRIMER_SEED_DIR` in the environment. The existing seed-discovery order in `primer-kb-load::auto_seed_if_empty` already prioritizes `$PRIMER_SEED_DIR`, so **zero changes to `primer-kb-load` are required**.

Detection algorithm:

```rust
fn resolve_packaged_seed_dir() -> Option<PathBuf> {
    let exe = std::env::current_exe().ok()?;
    let canonical = exe.canonicalize().ok()?;

    // Inside a .app: <something>.app/Contents/MacOS/primer-gui
    // We want:       <something>.app/Contents/Resources/<seed-subdir>
    let macos_dir = canonical.parent()?;       // .../Contents/MacOS
    if macos_dir.file_name()? != "MacOS" {
        return None;
    }
    let contents_dir = macos_dir.parent()?;    // .../Contents
    if contents_dir.file_name()? != "Contents" {
        return None;
    }
    let resources = contents_dir.join("Resources");

    // Search for *.jsonl under Resources/ at any depth Tauri may have placed
    // them. Tauri 2 preserves relative path structure with _up_ segments, so
    // the seed files land at Resources/_up_/_up_/_up_/_up_/data/seed/.
    find_jsonl_dir(&resources)
}
```

`find_jsonl_dir` walks `Resources/` via depth-first search to a bounded max depth (8 levels — comfortably exceeds Tauri's `_up_/_up_/_up_/_up_/data/seed/` = 5 levels), and returns the path of the first directory it visits that contains at least one `*.jsonl` file. DFS-first is acceptable because Tauri places all bundled resources under a single deterministic prefix; there is no realistic case where multiple `*.jsonl`-containing directories exist under `Resources/`. The bounded walk keeps startup latency negligible (~µs) even on cold disk.

`main.rs` calls this once at startup; if it returns `Some(path)`, `std::env::set_var("PRIMER_SEED_DIR", path)` runs before engine wiring. When running via `cargo run` from `src/` the canonical path is `src/target/{debug,release}/primer-gui` — the parent isn't `MacOS`, so `resolve_packaged_seed_dir` returns `None`, the env var stays unset, and the existing `CARGO_MANIFEST_DIR` fallback in `primer-kb-load` fires as today.

This logic lives in a small `paths` module inside `primer-gui` (not `primer-engine`, not `primer-kb-load`) — it is a GUI-specific concern tied to macOS `.app` layout, not a generic engine concern. The CLI never goes through this path.

### 4. Icons

Source: `assets/curious_childs_primer_icon.png` (1024×1024, RGBA, confirmed). Workflow:

```bash
cp assets/curious_childs_primer_icon.png src/crates/primer-gui/icons/source.png
cd src/crates/primer-gui
~/.cargo/bin/cargo tauri icon icons/source.png
```

`cargo tauri icon` generates the full required set: `icon.icns` (macOS), `icon.ico` (Windows — unused but generated), Linux PNGs, iOS/Android assets. The generated files are committed to the repo so the build script does not need to regenerate them.

A short note in `README.md` documents the source and regeneration command so future icon updates have a clear path.

### 5. `scripts/build-dmg.sh` (new file)

Single-purpose build driver. Fails fast on missing prerequisites. Lives at repo root (not under `src/`) because it's a project-wide tooling script, not a Rust artifact.

```bash
#!/usr/bin/env bash
set -euo pipefail

# Verify prerequisites
command -v cargo >/dev/null || { echo "cargo not found; install rustup" >&2; exit 1; }
~/.cargo/bin/cargo tauri --version >/dev/null 2>&1 || {
    echo "cargo-tauri not installed. Run:" >&2
    echo "  ~/.cargo/bin/cargo install tauri-cli --version '^2.0'" >&2
    exit 1
}

# Verify signing cert
if ! security find-identity -p codesigning -v | grep -q "Developer ID Application: Horst Herb (X5DWXB4283)"; then
    echo "Developer ID Application cert not found in keychain" >&2
    exit 1
fi

# Verify notarization credentials
for var in APPLE_API_ISSUER APPLE_API_KEY APPLE_API_KEY_PATH; do
    if [ -z "${!var:-}" ]; then
        echo "Missing notarization env var: $var" >&2
        echo "See README 'Building the macOS DMG' for setup." >&2
        exit 1
    fi
done
[ -r "$APPLE_API_KEY_PATH" ] || { echo "APPLE_API_KEY_PATH not readable: $APPLE_API_KEY_PATH" >&2; exit 1; }

# Build
cd "$(dirname "$0")/.."  # repo root
~/.cargo/bin/cargo tauri build \
    --manifest-path src/crates/primer-gui/Cargo.toml \
    --bundles dmg \
    --target aarch64-apple-darwin

DMG_PATH="src/target/aarch64-apple-darwin/release/bundle/dmg/Primer_0.1.0_aarch64.dmg"
[ -f "$DMG_PATH" ] || { echo "Expected DMG not produced at $DMG_PATH" >&2; exit 1; }

echo
echo "Built and notarized: $DMG_PATH"
echo "Test the install on another Mac before distributing."
```

`set -euo pipefail` is non-negotiable — bash defaults silently continue past failures.

## Data flow at runtime

```
Primer.app launches
    └─> primer-gui/src/main.rs
            ├─> resolve_packaged_seed_dir() walks Resources/, finds *.jsonl dir
            ├─> std::env::set_var("PRIMER_SEED_DIR", <path>)
            └─> tauri::Builder::default().setup(|app| { ... primer-engine wiring ... }).run()

primer-engine constructs the engine stack:
    └─> primer-kb-load::auto_seed_if_empty(kb, locale)
            └─> discover_seed_files(locale) reads $PRIMER_SEED_DIR FIRST
                    └─> finds bundled *.jsonl, loads all matching pack_id
```

The flow is identical to the CLI path; the only difference is where `PRIMER_SEED_DIR` comes from. CLI users either set it explicitly, rely on XDG, or let the `CARGO_MANIFEST_DIR` walk-up find the dev tree. Bundled GUI users get it set automatically before engine init.

## Testing

Three layers, all manual since they involve OS-level signing infrastructure that can't run in CI without distribution credentials.

**Unit test (Rust)**: `resolve_packaged_seed_dir` is pure modulo `current_exe()`. Factor it into a `resolve_packaged_seed_dir_from(exe_path: &Path, find_jsonl: impl FnOnce(&Path) -> Option<PathBuf>) -> Option<PathBuf>` so the path-walking logic is covered by unit tests with synthetic paths. Mock `find_jsonl` with a no-op closure for the structural test; cover the .app detection logic with explicit synthetic paths (real .app layout, dev-tree layout, completely unrelated path) and assert correct `Some`/`None` decisions.

**Smoke test (build)**: run `./scripts/build-dmg.sh` end-to-end on the dev machine. Verify the script fails clearly when any prereq is missing (cert, env vars, tauri-cli). Verify the produced DMG mounts, contains `Primer.app`, and the app passes `codesign --verify --deep --strict --verbose=4` and `spctl -a -t open --context context:primary-signature`.

**Smoke test (install)**: copy the DMG to a separate Mac (or a fresh user account on the dev Mac to simulate a non-developer environment), double-click, drag to Applications, launch. Verify:
- No Gatekeeper warning on first launch.
- Empty session-picker screen appears.
- Settings modal opens; backend selection works.
- After configuring a backend (cloud with API key, or Ollama at a reachable URL), a new session starts and the Socratic conversation produces useful output drawing on retrieved passages (i.e. the bundled seed corpus was loaded).
- Quitting and re-launching shows the past session in the picker, and resume works.

If the second-Mac install shows a Gatekeeper warning, notarization failed silently — investigate the bundler output and `xcrun notarytool log` for the submission ID.

## Risk and known unknowns

**Upward glob in `bundle.resources`**: Tauri 2's bundler may or may not accept `../../../../data/seed/*.jsonl`. The fallback (build.rs copies seed files into a crate-local `resources/` dir) is well-understood and adds ~10 lines. Risk: medium probability of falling back; low impact.

**Notarization rejection on first run**: Apple's notary can reject a binary for subtle entitlement or hardened-runtime nits (e.g. a bundled dylib that wasn't signed with the same identity, missing hardened-runtime flag). Mitigation: the notary returns a specific log line; iterate, add `disable-library-validation` if a dylib trips it, re-run. Each iteration costs ~3 min wall-clock (notary submission turnaround). Risk: medium probability of needing 1-2 iterations; low impact.

**Packaged-app seed-dir layout**: Tauri 2's `_up_` segment encoding for upward-pointing resources is documented but rarely exercised. If the runtime layout differs from the documented form, `find_jsonl_dir`'s bounded directory walk catches it regardless of where the JSONLs end up under `Resources/`. Risk: low.

**Build.rs OUT_DIR concerns**: if the build.rs fallback is needed, it must write into `OUT_DIR` (not into the source tree) to avoid polluting `git status`. Tauri's resource resolution must accept an absolute path from a build script. If it doesn't, the workaround is to copy the resources into a fixed subdir of the crate (e.g. `src/crates/primer-gui/resources/seed/`) and `.gitignore` that subdir. Risk: low.

**Cert renewal**: Developer ID Application certs expire after 5 years. When this cert expires, the build script will fail clearly at the `security find-identity` check. No silent regression possible.

## Sequencing

Single PR. Touch points are largely independent:
- `tauri.conf.json` + `entitlements.plist` + icon generation are config/asset changes.
- `main.rs` path resolution is a small, isolated module.
- `scripts/build-dmg.sh` is a leaf file.
- `README.md` is documentation.

The implementation plan should sequence so each piece is independently verifiable: icons first (visual check), then `main.rs` path resolution (unit test + `cargo run` smoke), then `tauri.conf.json` bundle config (unsigned `tauri build` smoke), then signing + notarization wiring (signed-build smoke), then `scripts/build-dmg.sh` (end-to-end), then README.

## Out-of-scope follow-ups

These belong in separate PRs/issues after this lands and is verified:

1. GitHub Actions macOS workflow that produces a notarized DMG release artifact on tag push.
2. Universal binary (`universal-apple-darwin`) once an Intel evaluator surfaces.
3. Speech mode in the bundle: requires bundling Whisper model + Piper voice + ensuring espeak-ng is available (either system install instruction or a vendored copy), plus `NSMicrophoneUsageDescription` and the `com.apple.security.device.audio-input` entitlement.
4. Hybrid retrieval in the bundle: requires either bundling the BGE-M3 ONNX model (~570 MB, blows DMG size up) or first-launch download UX (adds network dependency).
5. Auto-update channel (Tauri 2 supports it natively via the updater plugin).
6. Mac App Store submission path using the existing Apple Distribution cert. Adds sandboxing requirements, different entitlements, and Mac App Store review process.
