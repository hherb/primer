Issue #40 asked to aggregate per-source Wikipedia/Klexikon attribution: every article registered its **own** `sources` row (`wiki-simple:en:mercury`, …), so there was no shared parent row a credits UI could read for a single "Powered by Simple English Wikipedia" line. The issue offered three options; the owner chose **option 1** — keep the per-article rows (preserving per-article retrieval filtering + URLs) and add an optional `parent_source_id` self-FK pointing at one umbrella row.

Two sub-decisions, both resolved with the owner during the brainstorm:
- **Umbrella metadata produced by the Python ingest** (approach A), not derived in the Rust loader — the ingest is the only layer that knows the proper aggregated credit line (matches the "data-as-data, user-facing strings out of Rust" discipline).
- **Carried as a nested `parent_source` object on each passage** (approach A), not a separate header/sidecar record — keeps the JSONL a single record type, order-independent. The flat hand-drafted seed simply omits the field (it's `Option`).

What landed:
- **`primer-knowledge` schema v3 → v4.** `sources` gains nullable self-referential `parent_source_id TEXT REFERENCES sources(id)`. Fresh DBs get it from `create_sources_table`; existing v3 DBs get a guarded `ALTER TABLE … ADD COLUMN` (idempotent via `pragma_table_info`). `upsert_source`/`list_sources` carry the 6th column. **The FK is enforced** (`foreign_keys=ON`).
- **`primer-core`** — `SourceMeta` gains `parent_source_id: Option<String>` (`None` for hand-drafted seed and for umbrella rows themselves).
- **`primer-kb-load`** — `SeedPassage` gains optional nested `parent_source` (`ParentSource`) object. `load_jsonl` links each child source to its umbrella + registers the de-duped umbrella row, **upserting parents before children** to satisfy the FK.
- **`data/ingest`** — `to_passage` emits the umbrella `parent_source` for every Wikipedia-shaped passage (`id = "<id_prefix>:<pack_id>"`, aggregated attribution, site-root `source_url` via the new pure `_site_root` helper). Flat CC0 seed omits it.
- **Docs:** CLAUDE.md updated in three places (primer-knowledge schema-version bullet → v4; primer-kb-load bullet → parent_source/umbrella; data/ingest bullet → `_site_root` + `to_passage` emit). README/ROADMAP **untouched** — internal schema/pipeline quality change, no user-facing behaviour change, no milestone (neither file references attribution/sources).

**No corpus regeneration forced.** The field is optional end-to-end (`#[serde(default)]` in Rust, omitted by the seed JSONL), so the committed `wiki_passages.*.jsonl` keeps loading exactly as before; existing DBs gain the NULL column on next open with no rewrite. ## Patterns to reuse, not reinvent

New from this session:
- **When an issue is a "design decision" with the author's own enumerated options, surface those options to the owner via AskUserQuestion before designing** — don't silently pick. #40 listed 3 options + "not blocking"; confirming option 1 *and* the two sub-decisions (where umbrella metadata comes from; nested-object vs sidecar) took three quick questions and avoided building the wrong shape.
- **A nullable self-referential FK is the low-churn way to add a parent/umbrella relationship** to an existing flat lookup table: per-row rows stay filterable, the new column is `Option`/NULL for everyone who opts out, and the migration is a single guarded `ADD COLUMN`. No new table, no data rewrite.
- **An enforced self-FK forces an upsert *ordering*, not just a schema change.** `foreign_keys=ON` means parent rows must be written before children. The `HashMap` dedup loop's iteration order is arbitrary, so partition the writes: `for s in values().filter(|s| s.parent.is_none())` first, then the children. **TDD caught this** — the loader tests failed with `FOREIGN KEY constraint failed` against the naive single-loop version; the fix was an explicit two-pass write.
- **Carry optional structured data as `#[serde(default)] Option<NestedStruct>` to keep a JSONL/format consumer single-type and backward-compatible.** Old files (no field) deserialize fine; new files opt in; the parse loop never branches on record type. The flat seed corpus and the umbrella'd wiki layer share one `SeedPassage` shape.
- **Derive a site root with `urllib.parse.urlsplit` (`scheme://netloc/`), never by trimming a path suffix** — robust to ports and odd paths, and self-documents intent. Make it a tiny pure helper (`_site_root`) with its own unit tests rather than inlining the split at the call site.
- **When a golden/byte-exact fixture is downstream of a deterministic emitter you've independently unit-tested, regenerate the golden from the pipeline** (run `main` over the fixtures, write the output back) rather than hand-editing — the emitter's exact output is already pinned by the unit tests, so the golden just needs to reflect the new deterministic bytes. Diff it afterward to confirm only the intended field changed.

Carried forward (prior handoffs): two-kind classification when markers differ in *where* they may appear (issue #41); positional lead-anchor (`^.{0,N}?`) beats exact-title-match for "title-as-subject"; named const + `%`-format for a regex window; `Connection::prepare_cached(&stable_sql)` is the bulk-write idiom (pair with a per-instance precomputed-SQL struct); report a modest perf win honestly with the *why* and a comparative `#[ignore]` bench; in-memory SQLite isolates CPU from disk in a write bench; to find squash-merged branches safe to delete, match local names against `gh pr list --state merged --json headRefName`, not `git branch --merged`; a `backup/*` branch whose tip isn't in `main` is a deliberate artifact — surface it, don't delete it; split an over-500-line module into a directory module via `git mv` so blame stays put; model a multi-file asset bundle as N single-file `kind`s; clap `required_if_eq` is blind to default values (ArgGroup + runtime validator); a hidden GUI form field must still be sent by `gather()` if its DTO field is mandatory; `--features speech` clippy is invisible to `--workspace` clippy — run it explicitly when touching voice code; run cargo from `src/` so the 1.88 pin is honored.
