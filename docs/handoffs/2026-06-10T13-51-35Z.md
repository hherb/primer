## Patterns to reuse, not reinvent

New from this session:
- **Device validation surfaces ABI/contract bugs that no host or mock test can.** Three real `QnnBackend` bugs (callback-on-query, content-not-path, relative-path resolution) were all invisible to the 200+ host tests because the host path uses `MockGenieLibrary` тАФ they only appear against the real `libGenie.so`. When a backend has a real-library path the host can't exercise, **treat the first on-device run as a test phase**, and when it fails, get the *authoritative* contract (here: the QAIRT 2.45 headers + the reference app's source) before touching the FFI тАФ never guess at an ABI.
- **Fetch just the file you need from a huge remote zip via HTTP range-requests.** The QAIRT SDK is 1.66 GB; I needed three ~10 KB headers. `uv run --with remotezip` + a server that answers `206 Partial Content` (most S3/CDN do) pulls only the wanted entries. Confirm range support with `curl -r 0-0 -D -`.
- **The pure-function carve-out makes an unsafe-FFI fix testable.** The path-rewrite logic became `absolutize_genie_config(json, bundle_dir) -> Result<String>` тАФ no FFI, no I/O, 7 host unit tests тАФ even though the `open_dialog` call-site that uses it is device-only. Carve the testable logic out of the untestable boundary.
- **Diagnose a permission wall with the narrowest possible probe.** `exec 3</dev/fastrpc-cdsp` (a bare read-open) in BOTH the `shell` and the app-uid context isolated the blocker to the DSP node itself (not a lib, not the config) and proved it's an app-packaging boundary тАФ far cheaper than re-running the whole backend each time.

Carried forward (prior handoffs): a new inference-routing/resilience behavior is a decorator over `Arc<dyn InferenceBackend>`, built by `build_main_backend`, `name()` тЖТ primary, fall over ONLY pre-stream; rich per-turn signals reach a decorator via an `Option<T>` `GenerationParams` field; routing/scoring POLICY pure in `primer-core`, MECHANISM in `primer-inference`; over-500-lines-only-because-of-tests тЖТ `foo/{mod,tests}.rs`; mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (the `Update` DTO has no `#[serde(default)]`); run cargo from `src/` with `+1.88`; docs-only PRs are CI-path-ignored (#168) тЖТ admin-merge/direct-push.
