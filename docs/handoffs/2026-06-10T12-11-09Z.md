## Patterns to reuse, not reinvent

New from this session:
- **When a handoff doc lands a milestone, the living docs don't update themselves.** PR #212 committed a rich validation report but left ROADMAP/README/runbook/NEXT_SESSION saying the gate was still pending — a silent staleness that a reader of those docs would never catch. The handoff's own "Next steps" §3/§4 *were* the propagation TODO; a session that reads only NEXT_SESSION.md (stale) would miss it. **Always reconcile the git log against the brief at session start** — `git log --oneline -5` surfaced the un-propagated #212 commit that the brief predated.
- **Inline runbook corrections beat a wholesale rewrite when the original structure is sound but the facts are wrong.** The runbook's *shape* (prereqs → SDK → export → build → sideload → smoke → gate) was fine; only the *contents* were pre-hardware guesses. A `> ⚠️ CORRECTION (#n)` callout per stale block, plus a top banner, preserves the reader's mental model and makes each fix auditable against the handoff's numbered list — vs a rewrite that loses the "here's what we used to think" trail.
- **Verify a squash-merged branch is safe to delete by diffing it against `main`, not by trusting the merge state.** A squash-merge leaves the source branch "unmerged" by git's ancestry test (`git branch -d` refuses it). `git diff main <branch> --stat` returning empty is the real proof its content is in `main`; only then `git branch -D`.

Carried forward (prior handoffs): a new inference-routing/resilience behavior is a decorator over `Arc<dyn InferenceBackend>`, built by `build_main_backend`, `name()` → primary, fall over ONLY pre-stream (never mid-stream); rich per-turn signals reach a decorator via an `Option<T>` field on `GenerationParams` every other backend ignores; keep routing/scoring POLICY pure in `primer-core`, MECHANISM in `primer-inference`; when a decorator needs per-turn state the bare trait can't carry, own it INSIDE the decorator; a nudge term must sit BELOW the decision threshold with a drift-tripwire test; over-500-lines-only-because-of-tests → `foo/{mod,tests}.rs` (move tests out, don't fragment production); a pure-relocation refactor's TDD net is "existing tests pass identically before/after"; mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (`Update` DTO has no `#[serde(default)]`, so every new field is MANDATORY in every IPC payload + `settings.js::gather()`); run cargo from `src/` with `+1.88`; docs-only PRs are CI-path-ignored (#168) → BLOCKED → admin-merge/direct-push.
