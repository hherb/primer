**The key finding (corrects the prior handoff):** Gemma 2's GGUF carries `add_bos_token=true` and a chat template that literally begins with `{{ bos_token }}` — but llama.cpp's `apply_chat_template` renders that directive as **empty** (rendered head is `<start_of_turn>user`, not `<bos>`), deferring BOS insertion to the tokenizer's `add_bos` flag. So `template_embeds_bos=false`, the metadata leg fires, and exactly **one** BOS is prepended. This mirrors Llama 3 (whose literal `<|begin_of_text|>` is stripped the same way). **Consequence:** the literal-`<bos>`-in-template leg of the fix is **unreachable with any tested real model** and stays covered by the host unit tests in `params.rs` (`should_prepend_bos_skips_when_template_embeds_literal_bos`). The merged #255 metadata-gating fix is exactly what keeps Gemma correct.

## Patterns to reuse, not reinvent

- **The BOS diagnostic harness is the canonical way to verify any BOS-arch model.** One env-driven `cargo test` line prints the full `BosDecision` and optionally asserts (`PRIMER_LLAMACPP_EXPECT_PREPEND_BOS=true|false`). No eyeballing token ids.
- **Upstream vs Ollama GGUFs for llama.cpp testing:** Ollama maintains a divergent llama.cpp fork — its gemma3/gemma4 blobs do **not** load on `llama-cpp-2 0.1.146`. For Gemma, download an **upstream/HF** GGUF (`bartowski/gemma-2-2b-it-GGUF` loads clean). Qwen3 and Llama 3.x Ollama blobs *do* load fine.
- **Resolving an Ollama model to a GGUF path** (no download): read `~/.ollama/models/manifests/registry.ollama.ai/library/<model>/<tag>`, take the `model`-mediatype layer's `digest`, blob is `~/.ollama/models/blobs/sha256-<hash>` (note `-` not `:`). Symlink to a `*.gguf` name; `validate_gguf_path` only checks `is_file()`, llama.cpp loads by content magic.
- **Reading GGUF metadata without loading the model:** a ~30-line pure-Python GGUF KV reader (used this session to confirm `chat_template` starts with `{{ bos_token }}` and `add_bos_token=true`) is faster than a full model load when you only need to inspect template/tokenizer metadata.
- **llamacpp pure-helper pattern** (unchanged): feature-INDEPENDENT logic + unit tests go in `llamacpp/params.rs`; the `#[cfg(feature = "llamacpp")] mod real` in `engine.rs` consumes them.
- **Android device-debug recipe (task #1):** logcat is dead on this ROM. `adb -s 912607710061 shell run-as org.theprimer.gui cat files/recognizer.log`; invoke Tauri commands over WebView CDP (`adb forward tcp:9222 localabstract:webview_devtools_remote_<pid>` → `Runtime.evaluate("window.__TAURI__.core.invoke('<cmd>')", awaitPromise=True)`; helpers `/tmp/cdp_invoke.py`, `/tmp/cdp_dom.py`).
