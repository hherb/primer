### The proven split recipe (reuse it — worked cleanly again this session)
1. Baseline: `cargo test -p <crate>` green BEFORE touching anything; record the pass count.
2. Read the whole file; map natural responsibility boundaries.
3. Convert `foo.rs` → `foo/mod.rs` + sibling submodules (OR, for a crate root like `lib.rs`, just add `mod X;` sibling files next to it). `mod.rs`/`lib.rs` does `pub use submodule::{…}` for the external API so `crate::foo::Bar` paths are unchanged. When `foo/` already holds a `tests.rs`, you're just adding sibling `.rs` files.
4. **The `use super::*` private-import leak (handled cleanly again this session):** a flat module's private `use std::X;` / private helper fns are visible to its `#[cfg(test)] mod tests` via `use super::*`, but a `pub use` re-export façade does NOT carry private `use` aliases, and `pub use submodule::*` does NOT re-export a submodule's *private* items. Two fixes, applied together:
   - Bump any **cross-module helper fn/const/struct** (used by sibling submodules AND/OR tests) to `pub(crate)` in its submodule. Invisible to the external-pub-surface diff (`pub(crate)` ≠ `pub `). For a struct read across siblings, its **fields** also need `pub(crate)` (Rust field privacy is module-not-crate scoped for *siblings*; descendant modules of the struct's own module see private fields, siblings do not).
   - Add the **`use` aliases + submodule-internal items the façade can't carry** explicitly to `tests.rs`. This session that was `use crate::schema::table_exists;`, `use crate::vector::{blob_to_vec, vec_to_blob};`, and `use primer_core::embedder::Embedder;` (the trait moved out of `lib.rs`'s own `use` block because `lib.rs` no longer references it, so `super::*` stopped carrying it — the compiler names the missing trait for you).
5. Fix any moved relative paths: `super::sibling::…` that moved a level deeper, and `include_str!("../…")` → `include_str!("../../…")` when a file drops into a subdir (N/A this session — `lib.rs` stayed at crate root, siblings alongside it).
6. Verify behaviour-preserving: diff the `pub (struct|enum|fn|const|async fn|trait)` surface old-vs-new (must be empty), re-run the crate suite (count must match baseline), full-workspace `cargo test`, clippy `-D warnings`, fmt `--check`.

## Patterns to reuse, not reinvent

- **The production-split recipe** (6 steps above) — step 4's two-part `use super::*` fix is the one non-obvious trap; for a crate-root (`lib.rs`) split there is no `include_str!` depth bump (siblings stay at the same level). The sibling-struct-field `pub(crate)` requirement (LocaleSql this session) is a fresh wrinkle worth remembering: descendant modules see a struct's private fields, siblings do not.
- **Behaviour-preserving proof for a refactor = pub-surface diff + matching test count + clippy + fmt.** The pub-surface diff (`grep -oE 'pub (struct|enum|fn|const|async fn|trait) [A-Za-z_]+' | sort -u` old vs new, must be empty) is the cheap high-signal check that no external symbol got dropped or renamed. Include `async fn` (Tauri commands / async trait methods) and `trait`.
- **Check for `[features]` / `#[cfg(feature)]` before deciding the verify matrix.** `primer-knowledge` had none, so default `cargo test` sufficed. `primer-engine`/`primer-cli`/`primer-gui`/`primer-speech` DO — those splits need the feature-combos build too.
- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift. Cross-check documented *counts* against a fresh grep.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** (Wrong 4×.)
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths.
- **The machine clock is ~1 day behind the harness `currentDate`** (this session: `date -u` said 2026-07-01, harness said 2026-07-02). Use the harness date for human-readable text; the `date -u` output is fine for the handoff filename (it's just a monotonic archive key).
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time. (`cargo fmt` does NOT take the target lock, so it can overlap.)
- **A grep `-c` final pipe stage returns exit 1 on zero matches** — benign when the grep is the last stage and the real work passed; check the `Finished` / `test result: ok` line, not the pipeline exit code.
- **Branch each PR off `main`, not off the previous branch.**
