## What we shipped this session — `--languages` preference list (PR #196)

**Branch:** `feat/cli-languages-preference-list-21`. **PR:** #196 (open, base `main`). Branched from `main` @ `eafe3d7`. **Closes #21.**

**Problem:** `LearnerProfile.languages` (open-vocabulary ISO 639-1 preference list, ordered by preference) was silently seeded from `locale.pack_id()`, conflating it with the closed-enum bound `locale` that actually drives prompt-pack / speech / per-locale-KB dispatch. The two are semantically distinct — a German-UI child might also be comfortable in English — but the old code forced them equal.

**Resolution (issue #21 suggested approach):**
- **New pure `parse_languages(csv: Option<&str>, locale) -> Vec<String>`** in `primer-engine/src/learner.rs` — single left-to-right pass: split on `,`, trim, lowercase (ISO 639-1 convention), drop empties, dedup preserving first-occurrence order; falls back to `[locale.pack_id()]` when nothing survives.
- **`--languages <csv>` CLI flag** (`primer-cli`), defaulting to just the `--language` pack id (historical single-language behaviour).
- **`create_learner_with_id` now takes an explicit `languages: Vec<String>`** (5 call sites updated: 3 engine tests, GUI wiring, CLI). The GUI passes `parse_languages(None, cfg_locale)` to preserve current behaviour exactly — no GUI UI change (out of scope for the CLI-titled issue).

**Deliberate scope decisions (all confirmed with owner before implementing):**
- **Used verbatim — no force-prepend.** `--language de --languages en` stores `["en"]`, matching the issue's examples. Engine dispatch keys off `locale`, so divergence is harmless.
- **Open vocabulary — no closed-enum validation.** Nothing reads `languages` yet (documentation-as-data for a future content-language-hinting feature); values are not checked against `Locale`.
- **Fresh-learner only.** Seeds a new learner, exactly like the bound locale; `reconcile_persisted_learner` is untouched, so persisted wins on a returning child. (No edit path for an existing learner's languages — same limitation locale has today.)

## Patterns to reuse, not reinvent

New from this session:
- **Separate a bound dispatch key from an open preference list by parameterising the constructor, not deriving inside it.** `create_learner_with_id` used to derive `languages` from `locale` internally; lifting it to an explicit `languages: Vec<String>` param + a pure `parse_languages` helper made both the CLI flag and the GUI's "keep old behaviour" call one-liners, and made the parse rules unit-testable in isolation. The GUI call site documents *why* it passes the default (no UI field yet) so a future GUI-languages PR knows exactly where to wire in.
- **A pure CSV→list parser belongs in the shared engine crate, not the CLI.** Putting `parse_languages` in `primer-engine` (next to the other learner-wiring helpers) let both binaries reuse it and kept the 8 parse-rule tests out of the binary crate. Default-fallback-to-locale lives *in* the helper, so neither call site special-cases the empty input.
- **"Used verbatim, no force-prepend" is the right default when two fields are allowed to diverge.** The temptation was to ensure the bound locale always appears in the preference list; resisting it (engine keys off `locale`, not `languages`) kept the semantics honest and matched the issue's own examples (`--language de --languages en` → `["en"]`).

Carried forward (prior handoffs): brace-escape a placeholder validator like Rust `format!` (`{{`/`}}` literal, single-pass renderer — can't bolt onto `str::replace`); split an over-500-line module into a directory module via `git mv` so blame stays put; model a multi-file asset bundle as N single-file `kind`s to reuse single-file download infra; locale-independent asset cache when one model serves all locales; a pure mapping helper for a config gate keeps a Tauri-command decision unit-testable; `--features speech` clippy is invisible to `--workspace` clippy — run it explicitly; decouple orthogonal backend axes by injecting one into the other's builder (Option A); feature-gated constructor with a rebuild hint; GUI owns mirror config enums when the engine crate is an optional dep; one-shot config migration must be applied at the View layer too; clap `required_if_eq` is blind to default values (ArgGroup + runtime validator); free-form GUI text → parse in pure Rust; a hidden form field must still be sent by `gather()` if its DTO field is mandatory; stream-spanning text transforms need a stateful filter; `generate()` aggregates `generate_stream()`; user-facing strings route through `render_inference_error`; run cargo from `src/` so the 1.88 pin is honored.
