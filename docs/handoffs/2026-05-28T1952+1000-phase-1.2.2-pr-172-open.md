- **New module `primer-inference/src/qnn/`** in five files, all under the 500-line guideline:
  - `qnn/mod.rs` (33 lines) — re-exports `QnnBackend`, `MetaError`, `PrimerMeta`, etc.
  - `qnn/meta.rs` (374 lines) — `primer-meta.json` parser. Strict `load_from_bundle` raises a typed `MetaError`; lenient `load_or_fallback` synthesises ChatML defaults + bundle-dirname-derived id when the meta is absent. Forward-compat (`#[serde(default)]` on `stop_sequences`; unknown fields silently tolerated so a v2 schema doesn't brick old binaries).
  - `qnn/template.rs` (348 lines) — `minijinja`-backed Jinja2 chat-template renderer over `primer_core::inference::Prompt`. Flatten-and-prepend-system shape; pinned ChatML + Llama-3-Instruct fixtures; unicode/special-char passthrough verified.
  - `qnn/genie/{mod.rs (210), real.rs (306), mock.rs (125)}` — trait abstraction (`GenieLibrary` / `GenieDialog`) so the backend depends on a swap-able surface, not on the raw FFI. `genie::real::RealGenieLibrary` wraps `Arc<primer_qnn_sys::GenieLibrary>`. Construction is panic-safe via a stack-scoped config-handle guard between `config_create_from_json` and `dialog_create`. `Drop` releases dialog + config in the documented order. `genie::mock::MockGenieLibrary` (`#[cfg(test)]`) records every call so backend tests pin sequencing. `GenieCallError` is a structured enum (`PlatformUnsupported`, `LibraryLoad`, `SymbolMissing`, `ConfigNotFound`, `BadConfigPath`, `NonSuccess`) — variants chosen so the future i18n layer can route by shape, not by substring-matching a `String`.
  - `qnn/backend/{mod.rs (238), tests.rs (281)}` — `QnnBackend` struct + `InferenceBackend` impl. `new(bundle_dir, qairt_lib_dir)` is the production constructor (real library + smoke check on). `new_with_library(bundle_dir, Arc<dyn GenieLibrary>, run_smoke_check: bool)` is the test seam. **ABI smoke check at construction** (plan §1.2.2 task 8): one tiny throwaway query against the freshly-opened dialog confirms the FFI shape matches reality before the first conversation turn. `generate_stream` returns a single `TokenChunk{done:true}` for this step — per-token C-ABI callback bridge lands in step 1.2.3.

## Patterns to reuse, not reinvent

New from this session:

- **"Trait abstraction at the FFI seam, real + mock implementations under the same trait" is the cleanest test-driven pattern for unsafe FFI bindings.** `GenieLibrary` / `GenieDialog` traits give the backend a sync, mockable surface; the real impl wraps `Arc<primer_qnn_sys::GenieLibrary>` with the unsafe-block discipline; the `#[cfg(test)]` mock records every call and returns canned data. Same shape as `KnowledgeBase` / `LearnerStore` / `InferenceBackend` across the rest of the workspace, just narrower in scope (one library + one dialog).
- **Stack-scoped guard for paired C-API create/destroy** between two calls that can't be merged. `ConfigHandleGuard` in `genie::real` frees `GenieDialogConfig` on drop unless `defuse()` transfers ownership into the downstream owner (the dialog). Standard Rust idiom but worth pinning here because it's exactly what makes `open_dialog` panic-safe — if `dialog_create` panics or returns an error after `config_create_from_json` succeeded, the config still gets freed.
- **Structured error variants over `Other(String)` when an error will cross an i18n boundary.** `GenieCallError` has six variants (`PlatformUnsupported`, `LibraryLoad`, `SymbolMissing`, `ConfigNotFound`, `BadConfigPath`, `NonSuccess`) so a future i18n layer can pattern-match on the variant to choose a translated message. A single-variant `Other(String)`-style enum would force i18n to substring-match developer English, which is exactly the bug class the i18n boundary exists to prevent (same reasoning as PR #171's `GenieLibraryError` decision).
- **`#[serde(default)]` + tolerated unknown fields = forward-compatible config files.** Old binaries reading new meta files don't break on a field they don't know about; new binaries reading old meta files don't break on a field that's absent. The cost is that typos in field names become silent — mitigated here because each field is exercised by its own unit test.
- **"Convert single .rs file to directory with `mod.rs` + `tests.rs` sibling" is the cleanest way to keep an impl module under the 500-line guideline** when the inline tests outweigh the production code. Cargo finds `tests.rs` via `#[cfg(test)] mod tests;` from `mod.rs`, no `#[path]` trick needed. Used twice this session (`backend.rs` 514 → 238+281, and `genie.rs` 637 → 210+306+125 in a mod/real/mock trio).
- **"Verify the qnn-feature cross-compile locally before encoding it in CI" was attempted but blocked by no-local-NDK** — discipline preserved by deferring the CI step to a follow-up rather than landing-and-hoping. Same shape as PR #169's "verify the safety assumption first" before encoding a `paths-ignore` filter; same shape as PR #167's "verify the feature combo builds locally first" before adding the macOS feature-combos drift-guard.

Carried forward (from prior sessions; see prior handoff trail for two-layer drain, capacity-1-channel-as-sync-lever, pure-helper extraction, `#[path]` + `#[cfg(test)] mod` two-load pattern for build-script TDD, gate-narrowing, RFC-2229 whole-struct capture, trait-directive refusal documentation patterns, bundle-struct + free-function shape for shared closure builders, one-file-one-cargo-feature-gate as a quick file-size win, public-re-export protecting call sites from internal renames, generic-single-slot-cache + mechanical-caller-wiring as a TDD shape, `Option<T>` taken in finalize-AND-Drop for consume-on-success-AND-on-panic, put-back-before-propagate for caches keyed by expensive construction, "verify locally first, then encode in CI" for feature-combo drift-guards, "`cargo clippy -- -D warnings` over `cargo check` for drift-guards when the rotted-lint bug class exists", "verify the safety assumption before encoding a CI exclusion" for docs-only path filters, "document the branch-protection coupling at the point of the workaround", "document the swap path at the load-bearing line" for hand-rolled FFI bindings).
