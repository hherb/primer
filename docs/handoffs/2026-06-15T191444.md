## What we shipped this session (PR #240, branch `llamacpp-finish-reason-length`, closes #238)

`FinishReason::Length` now has **five producers, not four**. No dialogue-manager change — the recovery loop (`turn.rs`) was already backend-agnostic, so wiring the last producer enables the same locale-aware apology + progressive-shrink retry for llamacpp too.

| Backend | Native length signal | Mapping |
| --- | --- | --- |
| QNN | `GENIE_STATUS_CONTEXT_LIMIT_EXCEEDED` | `emit_query_outcome` |
| cloud (Anthropic) | `stop_reason: "max_tokens"` | `AnthropicEventTranslator` |
| Ollama | `done_reason: "length"` | `map_ollama_finish_reason` |
| openai-compat | `finish_reason: "length"` | `map_openai_finish_reason` |
| **llamacpp (this PR)** | **`max_tokens`/context budget exhausted without a natural stop** | **`LlamaEngine::infer -> Result<FinishReason>` seam** |

**Why llamacpp needed a seam change, not a pure mapper.** Unlike the HTTP backends (reason carried on a terminal JSON object), the llamacpp seam is a token callback + `infer()` — neither surfaced *why* generation stopped. So `LlamaEngine::infer` changed from `Result<()>` to `Result<FinishReason>`:
- `RealLlamaEngine::infer` (`engine.rs`, feature-gated) reports `Length` when the decode loop runs to its full `max_tokens` count without a natural stop (eos / matched stop sequence / consumer drop), else `Stop`. A `max_tokens` of 0 reports `Stop` (degenerate config, not a real truncation).
- `LlamaCppBackend::generate_stream` (`backend.rs`) stamps the reason onto the synthetic terminal chunk; `process_filtered_chunk` already threads `finish_reason` through the reasoning filter, so a truncated reply that opened with a `<think>` block keeps the flag.

## Open decisions / risks

- **PR #240 is open, not merged.** Merge once `cargo test (default features)` is green (host suite passed locally). 1 commit.
- **llamacpp needed a seam-signature change; the network backends did not.** The HTTP backends carry the finish reason on a terminal JSON object, so a stateless pure mapper sufficed; llamacpp's seam (`infer()` + token callback) carried no stop-reason at all, so `LlamaEngine::infer` had to return `Result<FinishReason>`. Both impls (`RealLlamaEngine` + the test `MockLlamaEngine`) were updated. **Do not "simplify" the seam back to `Result<()>`** — the backend can't otherwise know whether to stamp `Length` on the terminal chunk.
- **The `RealLlamaEngine` length logic is compile-checked only, not host-run.** Behavioural coverage is the mock (the `RealLlamaEngine` arm is `--features llamacpp`-gated and not exercised on the default test path). The budget-exhaustion-→-`Length` mapping is plausible (the loop runs to its full `max_tokens` count only when neither eos nor a stop sequence fired) but a real-GGUF smoke is the only thing that confirms llama.cpp's actual stop behaviour matches the assumption. Cheap optional follow-up (#238 residual).
- **`max_tokens` truncation on llamacpp is rarer than on QNN** (generous default budget + 8K+ windows on local models), same as the network backends — a robustness/parity fix, not a hot path. Still correct: it removes a silent mid-sentence stop.
- **Pedagogy on a 4B NPU model remains the headline open quality question** — technically excellent, pedagogically unverified at scale.

## Patterns to reuse, not reinvent

New/refined this session:
- **When a backend's streaming seam carries no native stop-reason (a token callback + a unit-returning `infer`), change the seam to return the `FinishReason`** rather than inventing a side channel. The blocking decode loop knows why it stopped: the idiomatic Rust signal is "assume `Length`, flip to `Stop` on any clean `break` (eos / stop sequence / consumer drop)", or equivalently track a `hit_natural_stop` flag. Update *both* the real impl and the test mock; the mock gains a builder (`MockLlamaEngine::truncated()`) so the truncated path is host-testable on the default `cargo test`.
- **The synthetic terminal/flush chunk is where the finish reason lands** — build it with the engine-reported reason, not `..Default::default()`, and rely on `process_filtered_chunk` to carry `finish_reason` through the reasoning filter (so `Length` survives a `<think>` strip).

Carried (still true from PR #237): a backend's terminal finish-reason maps via a small pure `map_*_finish_reason(native: &str) -> FinishReason` helper for HTTP backends (`"length"`/`"max_tokens"` → `Length`, `_` → `Stop`); when a backend splits the reason from the end-of-stream marker across two events (Anthropic), use a small stateful translator struct.

Carried (still true from PR #235): a backend→engine streaming signal = a defaulted field on the terminal `TokenChunk` (`FinishReason`), so adding a producer is local to one backend and needs no DM change; the reasoning filter must *carry* the field through, not rebuild with the default (commit `091d1a0`).

Carried (prior QNN/device handoffs, still true): the standalone `qnn_bench` example **can't reach the DSP** from a sideloaded/Termux process on this ROM — on-device throughput is instrumented **inside the APK** (read `<app_data>/.primer/qnn_metrics.jsonl` via `run-as cat`; opt-in via Settings → Diagnostics or `PRIMER_QNN_METRICS_PATH`); the metrics file is bounded (1 MiB + single `.1` backup, PR #229). Android scoped storage hides `adb`-pushed `/sdcard/Android/data/<pkg>` — stage app-internal via `adb push /data/local/tmp/<f>` → `run-as <pkg> cp`; this RedMagic ROM has **dead logcat + black screencap** — read app-internal files via `run-as <pkg> cat` + owner reads the screen; a reboot leaves `/data/user/0/<pkg>` encrypted until the owner enters the PIN; QNN APK rebuild = `--no-default-features --features qnn`; cargo from `src/` with `+1.88`; commits touching `.github/workflows` need `gh auth refresh -s workflow`. Android host facts: JDK 21 = Android Studio JBR for `JAVA_HOME`; NDK at `/opt/homebrew/share/android-ndk`.
