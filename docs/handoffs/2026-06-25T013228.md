## Patterns to reuse, not reinvent

- **Code licence ≠ weights licence for any HF model.** Always check the model-card `license:` tag + the repo `LICENSE` for the *weights* separately from the code repo's licence. Supertonic: MIT code, OpenRAIL-M weights. Same discipline applies to any future model dep.
- **Download-at-runtime keeps the AGPL repo clean of non-OSI weights.** Whisper, Piper, BGE-M3, and now Supertonic weights are all fetched at first use, never vendored. This is the structural reason ethical-source/RAIL weights don't conflict with the AGPL code distribution — keep it that way (never mirror weights into the repo).
- **"pure helpers ungated + model/hardware-needing path gated/`#[ignore]`'d"** remains the house pattern for backend-dependent tests.
- **The host-actionable backlog is thin right now** — most open issues are owner/hardware/SDK-gated (Android device, QAIRT login, mic-in-the-loop).
