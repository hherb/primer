## Patterns to reuse, not reinvent

New from this session:
- **Mirroring a CLI feature into the GUI is a 3-struct + wiring + frontend pattern.** (1) add the field to `BackendConfig` (persisted, struct-level `#[serde(default)]` → forward-compat for free), `BackendConfigView` (read DTO + its `From<&GuiConfig>` literal), and `BackendConfigUpdate` (write DTO + its `into_config` literal); (2) the Update DTO has **no `#[serde(default)]`**, so every new field is MANDATORY — grep every `GuiConfigUpdate` JSON test payload (find them via a field that's always present, e.g. `"reasoning_markers"`) and add the new keys, and make `settings.js::gather()` always send them; (3) swap the wiring call (`build_backend` → `build_main_backend`) and pass the field; (4) frontend = dom-ref + `populate()` + `gather()` + an optional reveal toggle, with HTML `id`s that exactly match the JS `getElementById` strings.
- **A `json!`-macro test payload tolerates trailing commas; a `r#"..."#` raw-string payload does not.** When adding a field to several payloads, check which kind each is — raw literals need the new field inserted as strict JSON.
- **Removing the last use of an import requires editing the `use` list too** (else clippy `-D warnings` fails on the unused import). `build_backend` was only used at one call site; swapping it to `build_main_backend` meant swapping the import, not just the call.
- **When CI is path-ignored for docs (#168) and a PR is kept open, commit the handoff to `main` via admin direct-push — never to the open feature branch** (a docs commit there stops the required check re-running on the new head and deadlocks the PR CLEAN → BLOCKED).

Carried forward (prior handoffs): a reliability fallback for any `Arc<dyn Trait>` is a GoF decorator, not a DM change (wrap primary+secondary, delegate; the async pre-stream/mid-stream split maps onto the `…().await` `Result` boundary; `name()` delegates to the primary); split a construction decision into a pure truth-table fn + thin glue; `Result::err().expect(...)` not `expect_err(...)` when `Ok` isn't `Debug`; a new pub fn in a sub-module needs an explicit crate-root re-export; adding a non-`Option` field to a shared struct breaks EVERY literal (grep `StructName {`); for a new in-process inference backend, the QNN/llamacpp trait-seam + mock + `spawn_blocking` + two-cfg `build_*_backend` free-fn is the template (reuse for RKNN); push the feature gate as deep as possible; a process-wide `OnceLock` init must read-back on `Err`; nullable self-FK for parent/umbrella; `#[serde(default)] Option<NestedStruct>` for backward-compatible JSONL; `Connection::prepare_cached(&stable_sql)` bulk-write idiom; a `backup/*` branch whose tip isn't in `main` is deliberate — surface, don't delete; clap `required_if_eq` is blind to defaults (ArgGroup + runtime validator); a hidden GUI form field must still be sent by `gather()` if its DTO field is mandatory; `--features X` clippy is invisible to `--workspace` clippy — run it explicitly; run cargo from `src/` with `+1.88`; GPU-variant cargo features must each chain to the base `llamacpp` feature in every crate.
