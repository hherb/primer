## Patterns to reuse, not reinvent

New from this session:
- **`tauri android init` does NOT add `[lib]`/`cdylib` for a binary-only crate.** Tauri mobile loads the app as a native shared lib (`MainActivity` → `loadLibrary`), so you must add `[lib] crate-type = ["staticlib","cdylib","rlib"]` yourself — and keep `rlib` so the existing `[[bin]]` + unit tests still build. The desktop-regression gate (build + tests green after the edit) is the thing to guard.
- **Tauri CLI forwards cargo args after `--`.** `cargo-tauri android build … -- --no-default-features` is how you reach the inner `cargo build` (the gradle/android-studio path doesn't expose features). This is how the Android build stays BM25-only (#157) without a gradle edit.
- **Tauri 2.11 NDK discovery ignores `NDK_HOME`** and scans `$ANDROID_HOME/ndk/<version>/` — symlink the Cask NDK in. And Gradle wants exactly JDK 21 (Android Studio's JBR on this host).
- **Subagent-driven execution caught the `[lib]` gap cleanly:** the Task 2 implementer reported it as a DONE_WITH_CONCERNS forward-looking note, which the controller folded into the Task 3 dispatch — the two-stage flow surfaced the real blocker before it became a build failure.

Carried forward (prior handoffs): a new inference-routing/resilience behavior is a decorator over `Arc<dyn InferenceBackend>`, built by `build_main_backend`, `name()` → primary, fall over ONLY pre-stream; routing/scoring POLICY pure in `primer-core`, MECHANISM in `primer-inference`; over-500-lines-only-because-of-tests → `foo/{mod,tests}.rs`; mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (the `Update` DTO has no `#[serde(default)]`); run cargo from `src/` with `+1.88`; docs-only PRs are CI-path-ignored (#168) → admin-merge/direct-push.
