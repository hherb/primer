An in-process, embedded **llama.cpp inference backend** (`LlamaCppBackend`) behind the non-default `llamacpp` cargo feature. The Primer can now run its Socratic conversation against a local GGUF in-process — no HTTP server (complements, doesn't replace, `OpenAiCompatBackend` which covers the out-of-process `llama-server` path). Binding: `llama-cpp-2` 0.1.146 (does NOT pull `ort`, so independent of the `=2.0.0-rc.10` pin).

**Architecture (mirrors the QNN trait-abstracted pattern so the orchestration is CI-covered):**
- A `LlamaEngine` seam, the `LlamaCppBackend` streaming bridge (mpsc + `spawn_blocking`), the reasoning-marker strip, and all pure helpers (`llamacpp/params.rs`) are **always compiled + host-tested on the default `cargo test`** via a `MockLlamaEngine`.
- ONLY `RealLlamaEngine`'s `llama-cpp-2` calls are `#[cfg(feature = "llamacpp")]`-gated. The engine emits **raw** tokens via an `on_token: &mut dyn FnMut(&str) -> bool` callback; the BACKEND strips reasoning (`<think>…</think>`) — so that integration is exercised on the default build.
- GGUF's embedded chat template via `model.apply_chat_template` (generic fallback if absent — no sidecar). `LlamaBackend::init()` once-per-process behind an init-race-tolerant `OnceLock`. Each `infer` makes a fresh `LlamaContext` (its `'a` lifetime borrows the model, can't be stored beside it) with a `Mutex<()>` serialising callers.

## Patterns to reuse, not reinvent

New from this session:
- **For a new in-process inference backend, the QNN backend is the template** (`crates/primer-inference/src/qnn/`): a trait seam (`GenieDialog` ↔ `LlamaEngine`) abstracting the blocking generation call, a mock impl for host tests, mpsc + `spawn_blocking` + a serialising mutex, and a two-cfg `build_*_backend` free fn in `primer-engine/src/wiring.rs` with a distinct "rebuild with --features X" hint. Reuse it wholesale for RKNN next.
- **Push the feature gate as deep as possible.** Gating only `RealLlamaEngine`'s body (not the whole module) let the trait + backend bridge + reasoning strip + pure helpers be tested on the DEFAULT `cargo test` via a mock — a strict improvement over QNN's whole-module gate (whose tests need `--features qnn`). The reasoning-strip integration in particular is genuinely CI-covered this way.
- **Put cross-cutting transforms (reasoning strip) in the always-compiled backend bridge, not the feature-gated engine.** The engine emits raw tokens via an `on_token` callback; the backend wraps them through `process_filtered_chunk`. The mock then drives the strip in CI.
- **A binding-version-drift task is the single fragile spot — isolate it behind the trait + feature and let the implementer reconcile against `cargo doc`.** Task 7 (the `llama-cpp-2` wrapper) was the only task where the plan's code needed signature fixes (u32 gpu-layers, `token_to_piece` shape); because it's the sole feature-gated file behind the `LlamaEngine` seam, none of the host-tested code was affected.
- **A process-wide one-shot init (`LlamaBackend::init()`) must read-back the populated cell on error, not propagate it.** Concurrent callers get `Err(AlreadyInitialized)` despite a healthy backend — store-on-Ok, ignore-on-Err, then `OnceLock::get().ok_or(...)`.
- **Subagent-driven execution cadence that worked:** trivial manifest/const tasks verified by reading the diff directly (no review subagents); the substantive core (trait+backend+mock) got a full independent review; feature-gated build-only tasks verified by build+clippy; one final holistic review over the whole branch caught the init-race. Batch tightly-coupled tasks (5+6) into one implementer.

Carried forward (prior handoffs): surface an issue-author's enumerated options to the owner before designing; nullable self-FK for parent/umbrella; `#[serde(default)] Option<NestedStruct>` for backward-compatible JSONL; `urllib.parse.urlsplit` for a site root; regenerate goldens from the deterministic emitter; two-kind classification when markers differ in where they may appear; `Connection::prepare_cached(&stable_sql)` bulk-write idiom; in-memory SQLite isolates CPU in a write bench; match local branch names against `gh pr list --state merged` to find safe-to-delete branches; a `backup/*` branch whose tip isn't in `main` is deliberate — surface, don't delete; `git mv` to split an over-500-line module; model a multi-file asset bundle as N single-file `kind`s; clap `required_if_eq` is blind to defaults (ArgGroup + runtime validator); a hidden GUI form field must still be sent by `gather()` if its DTO field is mandatory (no `#[serde(default)]` on `BackendConfigUpdate`); `--features X` clippy is invisible to `--workspace` clippy — run it explicitly; run cargo from `src/` with `+1.88` so the pin is honored.
