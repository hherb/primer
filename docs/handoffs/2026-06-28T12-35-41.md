## Patterns to reuse, not reinvent

- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol (e.g. `ProbeReasoning`) across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift. Cross-check any documented *counts* (test counts, crate counts, passage counts) against a fresh grep — counts rot silently as code grows.
- **A docs-only PR's Rust CI gates skip** (`.md` is in `paths-ignore`); only CodeQL + Cloudflare Pages run. Don't block on the Rust jobs for a markdown PR — they won't appear. Confirm via `gh pr checks <n>`.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** Sweep + detector commands in the resume block below.
- **A CI-config change verifies itself on its own PR.** Workflow-file edits are NOT in `paths-ignore`, so a new CI step runs on the PR that introduces it.
- **Before adding a CI step that compiles a feature stack, verify it green LOCALLY first** (`cargo clippy -p <crate> --features <X> --all-targets -- -D warnings`).
- **cpal on Linux needs `libasound2-dev`; macOS uses CoreAudio (no dep).** espeak-ng is a runtime data dir only, not a build/link dep.
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths. Always `git status --short` after a `git add`.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time, `tee` each, grep the captured file.
- **Branch each PR off `main`, not off the previous branch.**
