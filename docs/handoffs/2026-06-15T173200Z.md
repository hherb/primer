## Open decisions / risks

- **PR #237 is open, not merged.** Merge once `cargo test (default features)` is green (host suite passed locally). 1 commit.
- **Cloud needed a stateful translator; the other two did not.** Anthropic emits `stop_reason` in `message_delta` and `done` in `message_stop` ‚Äî two separate events ‚Äî so `AnthropicEventTranslator` carries the pending finish reason across events and stamps it on the terminal chunk. Ollama and openai-compat each put the finish reason on the same terminal object as `done`, so a stateless pure mapper suffices. **Do not "simplify" the cloud path back into `parse_anthropic_event`** ‚Äî that pure fn is stateless by design and can't see across events.
- **These are mapping-level + host-tested only.** No live cloud/Ollama/openai-compat server was hit this session (no API key burned, no local server spun up). The native length condition firing end-to-end against a real provider is unverified ‚Äî but the parse-layer wiring is the only new logic and it's fully covered. A real-provider smoke (force `max_tokens=8` on a cloud turn and confirm the apology+retry fires) is a cheap optional follow-up.
- **`max_tokens` truncation on cloud/Ollama/openai-compat is rarer than on QNN** because their `GenerationParams.max_tokens` is generous and their context windows are large ‚Äî so this is a robustness/parity fix, not a hot path. Still correct to have: it removes a silent mid-sentence stop.
- **Pedagogy on a 4B NPU model remains the headline open quality question** ‚Äî technically excellent, pedagogically unverified at scale.

## Patterns to reuse, not reinvent

New this session:
- **A backend's terminal finish-reason maps via a small pure `map_*_finish_reason(native: &str) -> FinishReason` helper** (`"length"`/`"max_tokens"` ‚Üí `Length`, `_` ‚Üí `Stop`), unit-tested in isolation, then called from the line/chunk parser. Mirror this for any future backend.
- **When a backend splits the finish reason from the end-of-stream marker across two stream events (Anthropic), use a small stateful translator struct** (`AnthropicEventTranslator { pending_finish }`) driven by the spawn loop: `observe` the reason on the metadata event, `stamp` it on the terminal chunk. Keep the underlying per-event pure parser unchanged so its existing tests stay green; the translator is the only stateful seam and is itself unit-tested by feeding an event sequence.

Carried (still true from PR #235): a backend‚Üíengine streaming signal = a defaulted field on the terminal `TokenChunk` (`FinishReason`), so adding a producer is local to one backend and needs no DM change; the reasoning filter must *carry* the field through, not rebuild with the default (commit `091d1a0`).

Carried (prior QNN/device handoffs, still true): the standalone `qnn_bench` example **can't reach the DSP** from a sideloaded/Termux process on this ROM ‚Äî on-device throughput is instrumented **inside the APK** (read `<app_data>/.primer/qnn_metrics.jsonl` via `run-as cat`; opt-in via Settings ‚Üí Diagnostics or `PRIMER_QNN_METRICS_PATH`); the metrics file is bounded (1 MiB + single `.1` backup, PR #229). Android scoped storage hides `adb`-pushed `/sdcard/Android/data/<pkg>` ‚Äî stage app-internal via `adb push /data/local/tmp/<f>` ‚Üí `run-as <pkg> cp`; this RedMagic ROM has **dead logcat + black screencap** ‚Äî read app-internal files via `run-as <pkg> cat` + owner reads the screen; a reboot leaves `/data/user/0/<pkg>` encrypted until the owner enters the PIN; QNN APK rebuild = `--no-default-features --features qnn`; cargo from `src/` with `+1.88`; commits touching `.github/workflows` need `gh auth refresh -s workflow`.
