## What we shipped this session

**Closed issue #125** — replace the raw GCD FFI bindings in [`primer-speech/src/macos/tts.rs`](src/crates/primer-speech/src/macos/tts.rs) with the [`dispatch2`](https://docs.rs/dispatch2/0.3.1/dispatch2/) crate.

The pre-#125 code carried two `unsafe extern "C"` items: the public `dispatch_async_f` function and the private `_dispatch_main_q` static. (The static was needed because `dispatch_get_main_queue()` is an inline C function with no stable export symbol.) The shape — `Box::into_raw` a context, hand it to a `extern "C" fn trampoline`, `Box::from_raw` inside the trampoline — is gone. The same one-shot dispatch onto the main queue is now a closure invocation: `DispatchQueue::main().exec_async(move || { ... })`.

The migration's only non-mechanical wrinkle is a Rust 2021 RFC 2229 disjoint-capture trap. `Retained<AVSpeechUtterance>` is `!Send` (objc2 marks `Retained<T>: Send` only when `T: Send + Sync`, which Apple's ObjC classes generally aren't). The pre-#125 code wrapped utterance + sender in `StreamCtx` + `unsafe impl Send for StreamCtx` so the whole struct could cross the trampoline via raw pointer. The migration keeps the same `StreamCtx` shape because the `!Send` problem is identical — but a closure that destructures `ctx` directly, or reads `ctx.utterance` / `ctx.tx` field-wise, inherits the per-field bound, and the per-field `Retained<AVSpeechUtterance>: !Send` defeats `exec_async`'s `F: Send + 'static` requirement. The fix is a `let owned = ctx;` identity-rebind on the first line of the closure body — that expression refers to `ctx` as a whole value, which forces whole-struct capture, and the closure becomes `Send` via the struct-level `unsafe impl`. Both the rationale and the warning ("do not simplify this away") are pinned in the in-file comments and in [CLAUDE.md:168](CLAUDE.md#L168).

## Patterns to reuse, not reinvent

New from this session:

- **Migrating raw `extern "C"` FFI to a safe wrapper crate often runs into Rust 2021 RFC 2229 disjoint capture** when the closure form replaces a function-pointer trampoline. If the original code wrapped state in a struct + `unsafe impl Send` to cross a thread boundary via raw pointer, the closure-form replacement must capture the whole struct (not its fields) for that unsafe impl to apply. The canonical fix is an identity-rebind (`let owned = original;`) on the first line of the closure body — that expression refers to the original as a whole value, which forces whole-struct capture. Destructuring the original inside the closure body OR referencing its fields directly is NOT enough — disjoint capture splits both. Pattern shows up wherever you replace `dispatch_async_f(queue, ctx, trampoline)` / `pthread_create(start_fn, arg)` / `CFRunLoopAddSource(loop, source, mode)` style C APIs with closure-based safe wrappers and the captured state holds anything `!Send` on its own.
- **`dispatch2::DispatchQueue::main().exec_async(...)` is the canonical macOS "submit one-shot work to the GCD main queue" idiom in safe Rust.** No need to roll the `dispatch_async_f` + private `_dispatch_main_q` symbol bindings yourself. The closure must be `FnOnce + Send + 'static`. Pair with `let owned = ctx;` (see above) when the captured state requires a struct-level `unsafe impl Send`.
