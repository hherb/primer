## Open decisions / risks

- **`9b6dfc7` is not pushed / no PR opened.** Outward-facing; left for owner authorisation (item #0).
- **The Kotlin + JNI surface is device-only-runtime-verifiable.** It cross-compiles and host-compiles clean, and the pure logic is `MockBridge`-tested, but the actual permission check + deep-link Intent only run on a device — that's item #1. Same shape as all prior Android-native device-gated work in this repo.
- **`open_app_settings` constructs a fresh bridge** (cheap — `new_jni_bridge` only borrows the cached `&'static JavaVM`); it does not reuse the loop's bridge. Fine because the bridge is stateless w.r.t. settings; revisit only if a future bridge gains per-instance state.
- **Auto re-request was intentionally NOT added** — the recovery affordance is the settings deep-link (user choice). MainActivity still fires the one-shot startup request; a denial there is what the new up-front check catches.

## Patterns to reuse, not reinvent

- **Trait-seam + MockBridge is the host-test path for any new device capability.** Add the method to `AndroidSpeechBridge`, give `MockBridge` a scriptable field/counter + a unit test, then the real JNI impl in `jni_bridge.rs` (cross-compile-verified only). Mirrors the qnn `GenieLibrary`/`GenieDialog` seam.
- **`dyn Trait` method calls need NO `use` of the trait** — calling `bridge.has_record_audio_permission()` on an `Arc<dyn AndroidSpeechBridge>` resolves via the trait-object type itself; importing the trait there is an unused-import warning. (Bit me once this session.)
- **Typed `StartVoiceModeError` variant + `voice.js` `err.kind` switch is the established voice-start error UX** — mirror `asset_missing`/`auto_download_disabled`. Handle the kind in BOTH `onToggleClick` and `tryAutoStartVoiceMode` (launch + session-switch auto-restore).
- **The `android` module in `primer-speech` is gated behind `--features android-native`** — a plain `cargo test -p primer-speech` does NOT compile/run its tests. Use `cargo test -p primer-speech --features android-native --lib android::` to exercise them.
- **Android device-debug recipe (unchanged):** logcat is dead on this ROM. `adb -s 912607710061 shell run-as org.theprimer.gui cat files/recognizer.log`; Tauri commands over WebView CDP (`adb forward tcp:9222 localabstract:webview_devtools_remote_<pid>` → `Runtime.evaluate("window.__TAURI__.core.invoke('<cmd>')", awaitPromise=True)`; helpers `/tmp/cdp_invoke.py`, `/tmp/cdp_dom.py`).
