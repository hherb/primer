## Patterns to reuse, not reinvent

- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** Sweep: `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`. Inline detector (loop over the sweep output, **excluding the `total` line**): `for f in $(... awk '$1>500 && $2!="total"{print $2}'); do awk '/#\[cfg\(test\)\]/{ln=NR} ln && NR==ln+1 && /^[[:space:]]*mod .*\{/{print FILENAME" inline mod @"NR}' "$f"; done`.
- **A CI-config change verifies itself on its own PR.** Workflow-file edits are NOT in `paths-ignore`, so a new CI step runs on the PR that introduces it — that PR run IS the Linux verification you can't do from a macOS host. Always confirm the new step shows `[success]` (not skipped) in the job's step list: `gh run view <id> --json jobs | python3 …`.
- **Before adding a CI step that compiles a feature stack, verify it green LOCALLY first** (`cargo clippy -p <crate> --features <X> --all-targets -- -D warnings`). Adding a red CI step is worse than the coverage gap.
- **cpal on Linux needs `libasound2-dev`; macOS uses CoreAudio (no dep).** Any CI job that builds a cpal-pulling feature on Linux must add it to the apt list. The existing `feature-combos-macos` job proves espeak-ng is NOT a build/link dep for the speech stack — it's a runtime data dir only, so a Linux speech-clippy step needs no espeak-ng.
- **Test-module extraction is the lowest-risk oversized-file cut — but it's exhausted.** The recipe (move `#[cfg(test)] mod <name> { … }` to a sibling file; replace with `#[cfg(test)]\nmod <name>;`) ran 28 PRs in a row (#268–#295). The next cut is production code with real visibility churn.
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths. `git add` from repo root needs the full path. Always `git status --short` after.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time, `tee` each, grep the captured file.
- **Branch each PR off `main`, not off the previous branch.**
