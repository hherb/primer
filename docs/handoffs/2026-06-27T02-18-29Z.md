## Patterns to reuse, not reinvent

- **Always run the full sweep before claiming a backlog is empty.** `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`, then for each detect an *inline* test module with `awk '/#\[cfg\(test\)\]/{getline; if($0 ~ /mod .*\{/) print FILENAME}'` (an inline `mod … {`, NOT an already-extracted `mod …;`). **Also count `#[cfg(test)]` markers — >1 means multi-module, use the N-file approach.**
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { … }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` reaches all parent privates (incl. cfg(test) helpers) with no visibility edits. Seventeen PRs in a row (#268–#284) used exactly this.
- **MULTIPLE sibling test modules → N separate files, one per module.** Each file's `super` is the parent crate-module, so every `use super::*` stays verbatim. Avoid nesting them all under one `parent/tests.rs` (that re-points `super` to `parent::tests`).
- **The test module name may NOT be `tests`.** Match the declaration (`mod <name>;`) and child filename to the actual name — grep the `mod …` line, don't assume `tests`.
- **A single `mod tests` may itself contain *nested* submodules.** That's still a clean single-FILE cut — move the whole outer module; the nested `use super::*` already pointed at `<parent>::tests`, which is unchanged.
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs` (crate root) submodule resolves to `src/<name>.rs`. A non-root module `foo.rs`'s submodule resolves to `src/foo/<name>.rs` — so `mkdir src/foo/` first. (consts.rs this session is a non-root module → `src/consts/tests.rs`, needed the `mkdir`.)
- **Use ABSOLUTE paths for `sed`/`head`/`mkdir`** — the Bash cwd resets between calls in this harness. Note: `git add` from the repo root needs the `src/`-prefixed path (`src/crates/...`), while `cargo`/`grep`/`sed` run from `src/` use `crates/...`. Always `git status --short` after.
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` — the only addition should be the `mod <name>;` line (the `#[cfg(test)]` attribute was already present above the inline `mod … {`, so it stays unchanged).
- **`cargo fmt` may leave one blank line above the first relocated line that `--check` rejects.** Delete it: `sed -i '' '1{/^$/d;}' <file>` (skip-safe — only removes a leading blank if present).
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline (consts 176, workspace 51 ok blocks) and confirm the identical count post-move. No new tests authored, none needed.
- **Extraction recipe used this session (verbatim):** `mkdir -p crates/<crate>/src/<mod>`; `sed -n '<inner_start>,<inner_end>p' <mod>.rs | sed 's/^    //' > <mod>/tests.rs` (dedent 4 spaces, inner body only — exclude the `mod tests {` and its closing `}`); `sed -i '' '<attr_line>,$d' <mod>.rs` then `printf '\n#[cfg(test)]\nmod tests;\n' >> <mod>.rs`. Run `cargo fmt -p <crate>` then `sed -i '' '1{/^$/d;}'` the new file.
- **Branch each PR off `main`, not off the previous refactor branch.**
