## Patterns to reuse, not reinvent

- **The BOS diagnostic harness is now the canonical way to verify #201 (or any future BOS-arch model).** One env-driven `cargo test` line (see task #1) prints the full `BosDecision` and optionally asserts. No need to eyeball token ids.
- **Resolving an Ollama model to a GGUF path** (no download): read `~/.ollama/models/manifests/registry.ollama.ai/library/<model>/<tag>`, take the `model`-mediatype layer's `digest`, and the blob is `~/.ollama/models/blobs/sha256-<hash>` (note `-` not `:`). Symlink it to a friendly `*.gguf` name; `validate_gguf_path` only checks `is_file()`, and llama.cpp loads by content (`GGUF` magic), not extension.
- **Which local models load on `llama-cpp-2 0.1.146`:** Qwen3, Llama 3.x load clean. Gemma-3/Gemma-4 from Ollama do **not** (fork divergence). Check before assuming a model is testable.
- **llamacpp pure-helper pattern** (unchanged): feature-INDEPENDENT logic + unit tests go in `llamacpp/params.rs`; the `#[cfg(feature = "llamacpp")] mod real` in `engine.rs` consumes them. Never put untested logic inside the feature-gated module.
