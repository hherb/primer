## Patterns to reuse, not reinvent

New from this session:
- **To find squash-merged branches safe to delete, match local branch names against `gh pr list --state merged --json headRefName`, NOT `git branch --merged`.** Squash merges rewrite history so the merged branch's tip is never an ancestor of `main` — `--merged` reports nothing and `--no-merged` reports everything, both useless. The PR's head-ref name is the reliable join key. `comm -12 <(sorted local) <(sorted merged-refs)` gives the safe set in one shot.
- **A branch named `backup/*` or `*-snapshot` whose tip isn't in `main` is a deliberate artifact — surface it, don't delete it.** Housekeeping deletes *merged* branches; a backup is kept precisely because it's *not* merged. Flag it for the owner instead of applying the bulk rule.
- **A `git branch -D` that fails with "used by worktree" means a stale worktree, not a stuck branch.** Check the worktree is clean (`git -C <path> status --short`), `git worktree remove <path>`, `git worktree prune`, *then* delete the branch. Don't force past the worktree.
- **Remote-branch deletion is outward-facing — confirm even inside an authorized "housekeeping" task.** The owner authorized local cleanup; the remote step modifies GitHub, so a one-line confirm was the right call before `git push origin --delete`.
