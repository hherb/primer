## Patterns to reuse, not reinvent

- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** Sweep: `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`. Inline detector (per file): `awk '/#\[cfg\(test\)\]/{ln=NR} ln && NR==ln+1 && /mod .*\{/{print FILENAME" inline mod @"NR}' <file>`. The prompt_builder/responsive misses both came from trusting stale notes.
- **Re-read each `#[cfg(test)]` marker's CONTEXT, not just the count.** A marker may sit above a `#[cfg(test)] fn helper` or a `#[cfg(test)] const` (left in place) or an already-extracted `mod tests;` — only `#[cfg(test)]\nmod <name> {` (inline brace) is extractable.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { … }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` (and `use super::SPECIFIC`) reach all parent privates with no visibility edits. 25 PRs in a row (#268–#292).
- **For FEATURE-gated files, the matching `--features` run IS the load-bearing guard** — a default `cargo test` won't even compile the module. Capture the pre-move test count under `--features X` and confirm the identical count post-move.
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs`/`mod.rs` (acts as a dir owner) submodule resolves to a sibling file in the SAME dir (no mkdir). A non-root `foo.rs`'s submodule resolves to `foo/<name>.rs` — `mkdir foo/` first (responsive_layout_contract.rs this session, and supertonic/macos26, are non-root → mkdir; the kb-load `common/mod.rs` pick is mod.rs-style → no mkdir).
- **Use ABSOLUTE paths for `sed`/`grep`/`mkdir`/`wc`** — the Bash cwd resets between calls (`cd src` failed twice this session). Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths under the repo. `git add` from repo root needs the `src/`-prefixed path. Always `git status --short` after.
- **Avoid concurrent `cargo test --workspace` runs** — two overlapping invocations collide on the target lock and one exits 1 (false alarm). Run the authoritative workspace test serially.
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` — the only addition should be `mod <name>;` (the `#[cfg(test)]` attribute was already present above the inline `mod … {`).
- **`cargo fmt` may leave one blank line above the first relocated line that `--check` rejects.** Delete it: `sed -i '' '1{/^$/d;}' <file>` (skip-safe).
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline and confirm the identical count post-move. No new tests authored, none needed.
- **Extraction recipe (verbatim, non-root flavour):** `mkdir -p crates/<crate>/src/<mod>` first; `sed -n '<inner_start>,<inner_end>p' <mod>.rs | sed 's/^    //' > crates/<crate>/src/<mod>/tests.rs` (dedent 4 spaces, inner body only — exclude `mod tests {` and its closing `}`); `sed -i '' '<attr_line>,$d' <mod>.rs` then `printf '#[cfg(test)]\nmod tests;\n' >> <mod>.rs`. Run `cargo fmt -p <crate>` then `sed -i '' '1{/^$/d;}'` the new file. **Then grep the new file for relative paths** (`include_str!`, `"../`) and adjust depth if the dir changed.
- **Branch each PR off `main`, not off the previous refactor branch.**
