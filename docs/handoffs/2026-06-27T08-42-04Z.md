## Patterns to reuse, not reinvent

- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** Sweep: `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`. Inline detector (loop over the sweep output, **excluding the `total` line**): `for f in $(... awk '$1>500 && $2!="total"{print $2}'); do awk '/#\[cfg\(test\)\]/{ln=NR} ln && NR==ln+1 && /^[[:space:]]*mod .*\{/{print FILENAME" inline mod @"NR}' "$f"; done`. The prompt_builder/responsive misses both came from trusting stale notes.
- **Re-read each `#[cfg(test)]` marker's CONTEXT, not just the count.** A marker may sit above a `#[cfg(test)] fn helper` or a `#[cfg(test)] const` (left in place) or an already-extracted `mod tests;` — only `#[cfg(test)]\nmod <name> {` (inline brace) is extractable.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod <name> { … }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod <name>;`. The test module is a *descendant*, so `use super::*` (and `use super::SPECIFIC`) reach all parent privates with no visibility edits. 28 PRs in a row (#268–#295) — now exhausted.
- **For FEATURE-gated files, the matching `--features` run IS the load-bearing guard** — a default `cargo test` won't even compile the module. Capture the pre-move test count under `--features X` and confirm the identical count post-move.
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs`/`mod.rs` (acts as a dir owner) submodule resolves to a sibling file in the SAME dir (no mkdir). A non-root `foo.rs`'s submodule resolves to `foo/<name>.rs` — `mkdir foo/` first.
- **Use ABSOLUTE paths for `sed`/`grep`/`mkdir`/`wc`** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths under the repo. `git add` from repo root needs the `src/`-prefixed path. Always `git status --short` after.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. This session, a `cargo test --workspace` launched concurrently with `cargo clippy --workspace` stalled (empty `tee` output) until clippy finished, then completed cleanly. Run cargo passes **one at a time**, `tee` each, then grep the captured file for both `FAILED` and the `ok` count.
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` — the only addition should be `mod <name>;` (the `#[cfg(test)]` attribute, deleted and re-appended identically, shows as unchanged).
- **`cargo fmt` may leave one blank line above the first relocated line that `--check` rejects.** Delete it: `sed -i '' '1{/^$/d;}' <file>` (skip-safe).
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline and confirm the identical count post-move. No new tests authored, none needed.
- **Extraction recipe (verbatim):** for a non-root file `mkdir -p crates/<crate>/src/<mod>` first; `sed -n '<inner_start>,<inner_end>p' <file> | sed 's/^    //' > <dest>/<name>.rs` (dedent 4 spaces, inner body only — exclude `mod <name> {` and its closing `}`); `sed -i '' '<attr_line>,$d' <file>` then `printf '#[cfg(test)]\nmod <name>;\n' >> <file>`. Run `cargo fmt -p <crate>` then `sed -i '' '1{/^$/d;}'` the new file. **Then grep the new file for relative paths** (`include_str!`, `"../`) and adjust depth if the dir changed.
