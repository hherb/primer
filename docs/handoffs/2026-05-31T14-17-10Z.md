## What we shipped this session — Supertonic 3 Stage C (decoupled STT/TTS)

**Architecture (Option A — inject the TTS):** the three voice-loop builders (`backends.rs`, `backends_macos_native.rs`, `backends_macos_native_26.rs`) no longer construct their TTS internally — each takes an injected `Arc<dyn StreamingTextToSpeech>` + `VoiceProfile`. STT picks *which builder skeleton* runs; TTS picks *which synthesiser is injected*; they vary independently. Two shared helpers in `primer-speech/src/voice_loop/selectors.rs` are the single construction path both binaries use: `build_tts(TtsBackend, &TtsAssets)` (feature-gated per arm; rebuild-hint error for an uncompiled choice) and `build_voice_backends(SttBackend, tts, voice, whisper_model, …)` (cfg-dispatches the STT builder).

### Surfaces

- **CLI:** `--tts piper|supertonic` (default piper) + `--supertonic-dir`/`--supertonic-voice-style`, portable build only. **Clap gotcha discovered:** `required_if_eq` is blind to default values, so the naive predicate would force piper assets on the plain REPL. Solved with an `ArgGroup` `tts_assets` (≥1 asset under `--speech`) + runtime `validate_speech_assets` per-TTS completeness check. **D2:** the macOS-native CLI build stays compile-time AVSpeech (Hindi needs the portable Whisper-STT build; macОS-native STT/TTS have no `hi-IN`).
- **GUI:** Settings → Speech now has **separate STT (Whisper / macOS Native) and TTS (Piper / Supertonic / macOS Native) dropdowns**, each disabled with a rebuild hint when its feature isn't compiled (`macos_native_speech_available` / `supertonic_tts_available`). **D3 migration:** the #189 coupled `SpeechSettings.backend` is now `Option<SpeechBackend>` (`skip_serializing`); `resolve_backends()` maps the legacy stack to the two halves when the new fields are at defaults, AND `From<&GuiConfig> for GuiConfigView` applies it so the modal shows — and saves — the user's real choice (without this a #189 macos-native user would silently revert to Whisper+Piper). The GUI defines its OWN `config::SttBackend`/`TtsBackend` mirror enums because `config.rs` is always-compiled but `primer-speech` is an optional dep; conversion happens at the `speech`-gated `voice/backends.rs` boundary. Asset resolver `resolve_voice_assets` is `(stt,tts)`-aware: whisper gated iff Whisper STT, piper iff Piper TTS, Supertonic never download-gated (paths come from the per-locale `SpeechLocaleOverride`).

## Patterns to reuse, not reinvent

New from this session:

- **Decouple two orthogonal backend axes by injecting one into the other's builder** (Option A), not by adding a sibling builder per combination. The three voice-loop builders take an injected `Arc<dyn Trait>`; a shared `build_X` helper constructs it feature-gated, and a `build_voice_backends` dispatcher selects the skeleton. Avoids an N×M builder explosion.
- **Feature-gated constructor with a rebuild hint** (`build_tts`): each arm is `#[cfg(feature)]` with a `#[cfg(not(feature))]` twin that returns a "rebuild with --features X" error. Mirrors the qnn pattern; lets the UI show-but-disable an option and fail clearly if selected on the wrong build.
- **GUI owns mirror config enums when the engine crate is an optional dep.** `config.rs` is always-compiled but `primer-speech` is `optional`, so the GUI can't name `primer_speech::...` there — define `config::SttBackend`/`TtsBackend` and convert at the `speech`-gated wiring boundary. (Same reason the GUI mirrors other backend enums.)
- **One-shot config migration must be applied at the View layer too**, not just at the wiring boundary — otherwise a `skip_serializing` legacy field shows defaults in the settings modal and the user silently loses their choice on save. `From<&GuiConfig> for GuiConfigView` applies `resolve_backends()`.
- **clap `required_if_eq` is blind to default values** → use an `ArgGroup` (≥1 of a set) at parse time + a runtime validator for the exact-set-per-mode check.
- **Adding a field to a whole-struct-cloned, `#[serde(default)]` config struct threads through View/Update DTOs for free** (unlike a non-default `*Update` field, which breaks every test JSON — the #188/#189 lesson). `SpeechSettings` clones whole into both DTOs.

Carried forward (prior handoffs): free-form GUI text → parse in pure Rust, frontend is a verbatim pass-through; a hidden form field must still be sent by `gather()` if its DTO field is mandatory; stream-spanning text transforms need a stateful filter; `generate()` aggregates `generate_stream()`; user-facing strings route through `render_inference_error`; feature-aware compile-time-conditional defaults; pure-core + thin-device-example for hardware-gated harnesses; run cargo from `src/` so the 1.88 pin is honored.
