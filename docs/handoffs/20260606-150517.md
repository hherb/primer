## Patterns to reuse, not reinvent

New from this session:
- **A benchmark for a new inference backend is "thin example + shared `bench` core", not a fresh harness.** The generic timing/aggregation/thermal/run machinery lives in `primer-inference::bench` (`metrics`/`prompts`/`thermal`/`run`); a new backend's bench is just a `required-features`-gated `examples/<backend>_bench.rs` that constructs the backend and calls `bench::run::{measure_prompt, format_report}` + `bench::evaluate`. Backend-specific acceptance targets live next to the backend (e.g. `qnn::bench::qnn_targets()`); a measurement-first probe just passes `BenchTargets::none()` (or builds from CLI flags).
- **All-`Option` `BenchTargets` + vacuous-pass `evaluate` is the one mechanism for both "hard gate" and "measurement-only".** `None` per criterion → vacuous pass; `is_measurement_only()` → the report prints numbers without a PASS/FAIL verdict and the example exits 0. Don't add a separate "measurement mode" flag — absence of targets IS the mode.
- **Extracting shared infra from a feature-gated module gives the pure logic default-`cargo test` coverage for free.** The QNN bench's percentile/report/verdict/thermal logic was `--features qnn`-only; moving it to an always-compiled crate-root `bench/` module put it on the default test path (and the shared corpus guard with it).
- **When you `cp` (not `git mv`) shared code out of a still-live module, keep every commit green on BOTH the default and the feature build** by creating the shared copy first (Task 1) and deleting the original only when you rewire the consumer (Task 3). git still detected the net change as a rename in the final diffstat.
- **A pure `format_report(...) -> String` (caller `println!`s it) beats a `print_report(...)` that writes stdout** — it's unit-testable without capturing stdout (assert on substrings).
- **`.expect("<invariant>")` over `.unwrap()` for a provably-infallible call in production code** — documents the invariant and honours the no-unwrap-in-production convention; the reviewer will flag a bare `.unwrap()` even when currently unreachable.

Carried forward (prior handoffs): mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (the `Update` DTO has no `#[serde(default)]`, so every new field is MANDATORY in every IPC test payload + `settings.js::gather()`); a reliability fallback for any `Arc<dyn Trait>` is a GoF decorator (wrap primary+secondary, delegate, `name()` delegates to the primary, fall back only at the pre-stream `…().await` `Result` boundary); split a construction decision into a pure truth-table fn + thin glue; a new pub fn in a sub-module needs an explicit crate-root re-export; adding a non-`Option` field to a shared struct breaks EVERY literal (grep `StructName {`); for a new in-process inference backend, the QNN/llamacpp trait-seam + mock + `spawn_blocking` + two-cfg `build_*_backend` free-fn is the template (reuse for RKNN); push the feature gate as deep as possible; a process-wide `OnceLock` init must read-back on `Err`; `#[serde(default)] Option<NestedStruct>` for backward-compatible JSON; `Connection::prepare_cached(&stable_sql)` bulk-write idiom; a `backup/*` branch whose tip isn't in `main` is deliberate — surface, don't delete; clap `required_if_eq` is blind to defaults (ArgGroup + runtime validator); `--features X` clippy is invisible to `--workspace` clippy — run it explicitly; run cargo from `src/` with `+1.88`; GPU-variant cargo features must each chain to the base `llamacpp` feature in every crate.
