## Patterns to reuse, not reinvent

- **Re-read each `#[cfg(test)]` marker's CONTEXT, not just the count.** A marker may sit above a `#[cfg(test)] fn helper` or an already-extracted `mod tests;` — neither is an inline module to move. Only `#[cfg(test)]\nmod <name> {` (inline brace) is extractable. The prompt_builder.rs miss this session came from trusting a stale "2 markers = multi-module" note.
- **Always run the full sweep before claiming a backlog is empty.** `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`, then for each detect an *inline* test module with `awk '/#\[cfg\(test\)\]/{getline; if($0 ~ /mod .*\{/) print FILENAME}'`.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { … }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` reaches all parent privates (incl. cfg(test) helpers) with no visibility edits. Twenty-four PRs in a row (#268–#291) used exactly this.
- **For FEATURE-gated files, the matching `--features` run IS the load-bearing guard** — a default `cargo test` won't even compile the module. Capture the pre-move test count under `--features X` and confirm the identical count post-move (qnn/metrics: 11 this session).
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs` (crate root) submodule resolves to `src/<name>.rs` (no mkdir). A non-root module `foo.rs`'s submodule resolves to `src/foo/<name>.rs` — `mkdir src/foo/` first (qnn/metrics.rs this session, and the two remaining picks, are all non-root → mkdir).
- **Use ABSOLUTE paths for `sed`/`head`/`mkdir`** — the Bash cwd may reset between calls. `git add` from the repo root needs the `src/`-prefixed path; `cargo`/`grep`/`sed` run from `src/` use `crates/...`. Always `git status --short` after.
- **Avoid concurrent `cargo test --workspace` runs** — two overlapping invocations collide on the target lock and one exits 1 (false alarm). Run the authoritative workspace test serially (this session ran it in the background, exit 0, 0 FAILED).
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` — the only addition should be `mod <name>;` (the `#[cfg(test)]` attribute was already present above the inline `mod … {`).
- **`cargo fmt` may leave one blank line above the first relocated line that `--check` rejects.** Delete it: `sed -i '' '1{/^$/d;}' <file>` (skip-safe).
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline and confirm the identical count post-move. No new tests authored, none needed.
- **Extraction recipe used this session (verbatim, non-root flavour):** `mkdir -p crates/<crate>/src/<mod>` first; `sed -n '<inner_start>,<inner_end>p' <mod>.rs | sed 's/^    //' > crates/<crate>/src/<mod>/tests.rs` (dedent 4 spaces, inner body only — exclude `mod tests {` and its closing `}`); `sed -i '' '<attr_line>,$d' <mod>.rs` then `printf '\n#[cfg(test)]\nmod tests;\n' >> <mod>.rs`. Run `cargo fmt -p <crate>` then `sed -i '' '1{/^$/d;}'` the new file.
- **Branch each PR off `main`, not off the previous refactor branch.**
