## Patterns to reuse, not reinvent

- **Always run the full sweep before claiming a backlog is empty.** `find src/crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'`, then for each detect an *inline* test module with `awk '/#\[cfg\(test\)\]/{getline; if($0 ~ /mod .*\{/) print FILENAME}'` (an inline `mod … {`, NOT an already-extracted `mod …;`). The prior brief skipped this and was wrong.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { … }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` reaches all parent privates (incl. cfg(test) helpers) with no visibility edits. Fifteen PRs in a row (#268–#282) used exactly this.
- **MULTIPLE sibling test modules → N separate files, one per module** (this session's recipe). Each file's `super` is the parent crate-module, so every `use super::*` stays verbatim. Avoid nesting them all under one `parent/tests.rs` (that re-points `super` to `parent::tests` and forces a glob-of-glob gamble).
- **The test module name may NOT be `tests`.** cloud.rs's second was `classify_tests`; wiring's seven all had bespoke names. Match the declaration (`mod <name>;`) and child filename to the actual name — grep the `mod …` line, don't assume `tests`.
- **A single `mod tests` may itself contain *nested* submodules (ollama.rs has `classify_ollama_tests` + `reasoning_wiring`).** That's still a clean single-FILE cut — move the whole outer module; the nested `use super::*` already pointed at `<parent>::tests`, which is unchanged.
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs` (crate root) submodule resolves to `src/<name>.rs`. A non-root module `foo.rs`'s submodule resolves to `src/foo/<name>.rs` — so `mkdir src/foo/` first. This session: `cloud.rs` → `src/cloud/*.rs`; `wiring.rs` → `src/wiring/*.rs`.
- **Check `#[cfg(feature …)]` gating of the MODULE.** Default-build files use a plain `cargo test -p <crate> --lib` guard; feature-gated ones (supertonic, qnn/metrics, macos26) need the matching `--features` run as the load-bearing guard.
- **Use ABSOLUTE paths for `sed`/`head`/`mkdir`** — the Bash cwd resets between calls in this harness. Always `git status --short` after.
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` — the only additions should be the `mod <name>;` lines (fmt may reorder them alphabetically — fine).
- **`cargo fmt` leaves one blank line above the first relocated line that `--check` rejects.** Delete it per file: `sed -i '' '1{/^$/d;}' <file>`. (Skip-safe — only removes a leading blank if present; harmless on files starting with `//!`.)
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline (inference 199, engine 72, workspace 51 ok blocks) and confirm the identical count post-move. No new tests authored, none needed.
- **Branch each PR off `main`, not off the previous refactor branch.** If you accidentally branch off a sibling refactor branch, fix it with `git stash && git checkout main && git branch -D <branch> && git checkout -b <branch> && git stash pop` before committing.
