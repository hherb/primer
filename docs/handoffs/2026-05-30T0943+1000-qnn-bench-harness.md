## Open decisions / risks

- **The bench numbers are unmeasured.** The harness *encodes* the targets and the verdict, but nothing has run on a Qualcomm device. Do NOT report "QNN hits 15 tok/s" or "passes thermal" — only "the harness is built and host-tested." The numbers are the deliverable of step 1.2.0, not this session.
- **Token counting is chunk-based.** `decode_tokens` counts stream chunks after the first. This is an *exact* token count only if Genie fires its callback once per decoded token (which it does for the QNN path). If a future backend batches multiple tokens per chunk, the tok/s figure becomes a chunk/s under-count — documented in `metrics.rs`. Re-check when the real device callback cadence is observed in 1.2.0.
- **`BENCH_MAX_TOKENS=128` and `DEFAULT_DURATION_SECS=900` are first-guess defaults.** 128 tokens/prompt gives a stable steady-state decode window; 900s drives the SoC to thermal steady-state. Both are CLI-overridable. If the device run shows decode is dominated by prefill at 128 tokens, bump the cap.
- **`DEFAULT_PROMPTS_PATH = "../data/bench/socratic_prompts.jsonl"`** is relative to the `src/` working dir the example is launched from (matches the documented `cd ~/primer/src` invocation). Launching from elsewhere needs an explicit `--prompts`.
- **Decode/TTFT gate choice (min + p95) is deliberately conservative.** A run that passes passes for the worst representative prompt, not on average. If the device run shows a single pathological prompt dragging the verdict, revisit whether p95/min are the right aggregations (documented in `evaluate`'s doc comment).
- (… plus carried-forward risks: ort-sys vendor liability, pykeio anti-AI-PR policy, small-context budget untested on a real 4K tokeniser, QNN ABI smoke unverified vs real libGenie.so. See prior handoffs under `docs/handoffs/`.)

## Patterns to reuse, not reinvent

New from this session:

- **"Pure core + thin device example" makes a device-only deliverable host-testable.** Everything computable without the NPU (percentiles, pass/fail, CSV, sysfs parsing, prompt loading) is a pure function with unit tests; the example carries only I/O orchestration. Result: a benchmark that *can't fully run on host* still has ~35 host tests covering all its logic. Default to this shape for any hardware-gated harness.
- **A shipped data file gets a `#![cfg(feature=…)]` corpus regression test.** `tests/bench_corpus.rs` loads the real `data/bench/socratic_prompts.jsonl` via `CARGO_MANIFEST_DIR` + `../../../`, asserts the plan's 30-prompt floor and unique labels. A data file with no test silently rots.
- **`required-features` on a `[[example]]` keeps `cargo build --examples` green on default builds** while letting the example use a feature-gated type. CI builds it explicitly under the feature as the drift guard.
- **Conservative verdict aggregations (min for sustained, p95 for tail) beat means for acceptance gates** — a mean hides the one prompt that throttles.
