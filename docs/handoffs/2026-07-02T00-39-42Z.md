### The proven split recipe (reuse it — worked cleanly again this session)
1. Baseline: `cargo test -p <crate>` (with the right `--features` if the tests are gated) green BEFORE touching anything; record the pass count.
2. Read the whole file; map natural responsibility boundaries.
3. Convert `foo.rs` → `foo/mod.rs` + sibling submodules (OR, for a crate root like `lib.rs`, add `mod X;` siblings). When `foo/` already holds `*_tests.rs`, `git mv foo.rs foo/mod.rs` and add production siblings alongside. `mod.rs` does `pub use submodule::{…}` for the external API so `crate::foo::Bar` paths are unchanged.
4. **The `use super::*` private-import leak (handled cleanly again this session):** a flat module's private `use std::X;` / `use crate::Y;` aliases are visible to its `#[cfg(test)]` modules via `use super::*`, but a `pub use` re-export façade does NOT carry those private `use` aliases, and `pub use submodule::*` does NOT re-export a submodule's *private* items. Two fixes, applied together:
   - Keep any **private helper/const the tests reach via `super::*`** (this session: `resolve_qairt_lib_dir_for`, `PathBuf`) IN `mod.rs`, so the test module (a child of `mod.rs`) still sees it directly. Alternatively bump it to `pub(crate)` and add an explicit `pub(crate) use` re-export.
   - Add the **`use` aliases the façade can't carry** explicitly to the affected test files. This session: `ClassifierSettings`/`ExtractorSettings`/`InferenceBackend`/`Arc` — the compiler names each missing symbol for you. Gotcha: `.name()` on `Arc<dyn Trait>` does NOT need the trait imported (the method is intrinsic to the trait object), so don't reflexively add the trait import — it'll be flagged unused.
5. Fix any moved relative paths: `super::sibling::…`, and `include_str!("../…")` → `include_str!("../../…")` when a file drops into a subdir (N/A this session — siblings stayed at the same level).
6. Verify behaviour-preserving: diff the `pub (struct|enum|fn|const|async fn|trait)` surface old-vs-new (must be empty), re-run the crate suite (count must match baseline), **feature-combos clippy if the file has cfg arms**, full-workspace `cargo test`, clippy `-D warnings`, fmt `--check`.

## Patterns to reuse, not reinvent

- **The production-split recipe** (6 steps above) — step 4's two-part `use super::*` fix is the one non-obvious trap. This session's fresh wrinkles: (a) keep private helpers the tests reach (`resolve_qairt_lib_dir_for`) IN `mod.rs` rather than a submodule, so the test child still sees them via `super::*` with zero re-export churn; (b) `.name()` on `Arc<dyn Trait>` needs no trait import — don't add one reflexively (it'll be unused-flagged).
- **Behaviour-preserving proof for a refactor = pub-surface diff + matching test count + clippy + fmt (+ feature-combos clippy if cfg arms exist).** The pub-surface diff (`grep -oE 'pub (struct|enum|fn|const|async fn|trait) [A-Za-z_]+' | sort -u` old vs new, must be empty) is the cheap high-signal check that no external symbol got dropped or renamed. Include `async fn` and `trait`. Note `pub use foo::*` re-export lines do NOT pollute the diff (the regex requires `pub struct|enum|fn|…`, not `pub use`).
- **Check for `[features]` / `#[cfg(feature)]` before deciding the verify matrix.** `primer-knowledge` (#304) had none, so default `cargo test` sufficed. `primer-engine` (this PR), `primer-cli`, `primer-gui`, `primer-speech` DO — those splits need the feature-combos build too. `primer-engine`'s cfg features: `embedding`, `ollama-embedding`, `openai-compat-embedding`, `qnn`, `llamacpp` (+ GPU variants).
- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** (Wrong 4×.)
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths. `git show HEAD:<path>` wants the **repo-root-relative** path (`src/crates/…`), even when cwd is `src/`.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time. (`cargo fmt` does NOT take the target lock, so it can overlap.)
- **A grep `-c` / final pipe stage returns exit 1 on zero matches** — benign when the grep is the last stage and the real work passed; check the `Finished` / `test result: ok` line, not the pipeline exit code.
- **Branch each PR off `main`, not off the previous branch.**
