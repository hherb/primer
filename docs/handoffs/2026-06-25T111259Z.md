## Patterns to reuse, not reinvent

- **Always run the full sweep before claiming a backlog is empty.** The prior handoff's "exhausted" claim came from surveying only 4 files. The correct survey is `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'` then `grep -c '#[cfg(test)]'` each to find the ones with a big inline test module. Do this first.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { … }` into a sibling `<module>/tests.rs` and replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` reaches all parent privates (incl. cfg(test) helpers) with no visibility edits. Six PRs in a row (#268–#273) used exactly this. Prefer it over moving production code down (which costs `pub(super)`/`pub(crate)` churn).
- **Verify the marker count before the `sed` cut.** `grep -c '#[cfg(test)]'` — exactly `1` means a single clean trailing module; `>1` means there may be standalone `#[cfg(test)] fn` helpers ABOVE the module that a naive `sed -n 'START,ENDp'` would either miss or wrongly capture. For `>1` files, find every marker line and confirm the boundary by hand.
- **Relocate blocks with `sed` line ranges, never hand-transcription.** A ~1000-line verbatim move is exactly where Edit/Write silently corrupts a line. `head -N` to trim the parent, `sed -n 'A,Bp' > child.rs` to extract the module *body* (inner lines only — drop the `mod tests {` wrapper and closing brace), then `cargo fmt` to dedent. **Confirm the move is clean with `git diff --cached -- parent.rs | grep '^+' | grep -v '^++'`** — the only addition to the parent should be `mod tests;`.
- **`cargo fmt` may leave one blank line above the first relocated line that `--check` then rejects.** After moving a module body, run `cargo fmt --all -- --check` and delete the leading blank line if flagged (`sed -i '' '1{/^$/d;}'`). One-line fix; don't be surprised by it.
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline (here: `cargo test -p primer-gui --lib commands::session` = 28) and confirm the identical count passes post-move under the new module path. No new tests authored, none needed.
- **Check `cfg(feature …)` in the moved span AND the crate's default features before trusting a default `cargo test`.** A `speech`/`macos-native-26`/etc.-gated crate or span needs the feature matrix.
