## Patterns to reuse, not reinvent

New from this session:
- **A reliability fallback for any `Arc<dyn Trait>` is a GoF decorator, not a DM change.** Wrap primary+secondary, implement the same trait, delegate. The async pre-stream/mid-stream split maps *exactly* onto the `â€¦().await` `Result` boundary â€” `Ok` past that point is returned verbatim so downstream errors propagate without a second attempt. `name()` must delegate to the primary if any downstream logic keys off it (here: `is_small_context_backend`).
- **Split a construction decision into a pure truth-table function + thin glue.** `plan_main_backend(bool, bool, bool) -> enum` is exhaustively testable with zero I/O; the glue just builds the legs and materializes the plan. Test the glue with stub (always-builds) + `unknown-*` (always-fails) backend names for observable `name()`/error outcomes.
- **`Result::err().expect(...)` not `expect_err(...)`** when the `Ok` type isn't `Debug` (e.g. `Arc<dyn InferenceBackend>`): `expect_err` needs `T: Debug`; `err()` drops the `Ok` value first.
- **A new pub fn in a sub-module needs an explicit crate-root re-export** before another crate can import it (`build_main_backend` was added to `wiring.rs` but the CLI imports `primer_engine::build_main_backend` â€” surfaced during subagent execution, fixed in Task 6).
- **Adding a non-`Option` field to a shared struct (`BackendParams`) breaks EVERY literal** â€” grep `StructName {` and update the CLI site, the GUI site, AND all `#[cfg(test)]` constructors (struct-update `..base()` forms inherit automatically).
- **Subagent-driven TDD cadence worked well:** fresh implementer per task (sonnet) + spec-then-quality review (sonnet), final whole-impl review (opus). Combine spec+quality into one ordered reviewer for trivial pure-function tasks to save round-trips; keep them separate for integration glue.
- **Handoff placement when a PR is kept open:** commit NEXT_SESSION.md + handoff to **main** (admin direct-push), never to the open feature branch (would deadlock its CI).

Carried forward (prior handoffs): for a new in-process inference backend, the QNN/llamacpp trait-seam + mock + `spawn_blocking` + two-cfg `build_*_backend` free-fn is the template (reuse for RKNN); push the feature gate as deep as possible; cross-cutting transforms in the always-compiled bridge; a process-wide `OnceLock` init must read-back on `Err`; nullable self-FK for parent/umbrella; `#[serde(default)] Option<NestedStruct>` for backward-compatible JSONL; `Connection::prepare_cached(&stable_sql)` bulk-write idiom; a `backup/*` branch whose tip isn't in `main` is deliberate â€” surface, don't delete; clap `required_if_eq` is blind to defaults (ArgGroup + runtime validator); a hidden GUI form field must still be sent by `gather()` if its DTO field is mandatory; `--features X` clippy is invisible to `--workspace` clippy â€” run it explicitly; run cargo from `src/` with `+1.88`; GPU-variant cargo features must each chain to the base `llamacpp` feature in every crate (verify with `cargo tree â€¦ | grep primer-engine`).
