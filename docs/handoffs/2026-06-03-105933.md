## Patterns to reuse, not reinvent

New from this session:

- **Model a multi-file asset bundle as N single-file `kind`s, not one bundle kind**, when the download infra is single-file — you reuse resume/oversize/timeout unchanged and get per-file resume granularity for free. A locale-independent const table (`SUPERTONIC_ASSETS`) with a `slot` enum maps each file to its on-disk location; a drift test pins kind-string identity between the table and the IPC `kind` constants.
- **Locale-independent asset cache** (`supertonic/`) when one model serves all locales — distinct from the per-locale `voice/<locale>/` pattern. Resolve via an override-or-default helper, not the per-locale `LocaleDefault` table.
- **A pure mapping helper for a config gate** (`missing_to_error(flag, AssetMissing) -> Error`) keeps a Tauri-command-level decision unit-testable without the command. Pair a new serde-tagged error variant with a frontend branch that reuses the existing error-surface (no new modal).
- **`--features speech` (and other feature-gated) clippy is invisible to `--workspace` clippy** — a latent `field_reassign_with_default` lived in speech-gated test code for a whole release. When touching feature-gated code, run `clippy -p <crate> --features <feat> --all-targets -- -D warnings` explicitly. Prefer struct-update syntax (`X { field: v, ..Default::default() }`) over `let mut x = default(); x.field = v;` in tests.

Carried forward (prior handoffs): decouple orthogonal backend axes by injecting one into the other's builder (Option A), not an N×M builder explosion; feature-gated constructor with a rebuild hint; GUI owns mirror config enums when the engine crate is an optional dep; one-shot config migration must be applied at the View layer too; clap `required_if_eq` is blind to default values (use an ArgGroup + runtime validator); free-form GUI text → parse in pure Rust; a hidden form field must still be sent by `gather()` if its DTO field is mandatory; stream-spanning text transforms need a stateful filter; `generate()` aggregates `generate_stream()`; user-facing strings route through `render_inference_error`; run cargo from `src/` so the 1.88 pin is honored.
