## What we shipped this session — i18n brace-escaping (PR #195)

**Branch:** `i18n/placeholder-brace-escaping-20`. **PR:** #195 (open, base `main`). Branched from `main` @ `6f60994`. **Closes #20.**

**Problem:** `primer_pedagogy::prompt_pack::validate_placeholders` rejected any `{ident}` token not on a per-field allowlist. That catches real typos (`{nme}`) but also rejected narrative a translator might want literally (`{Beispiel}`, `{ejemplo}`, `{a}`) — a hard `PrimerError::Config` at startup. Surfaced in PR #17 review; a risk for the German pack and `PRIMER_PROMPTS_DIR` translator iteration.

**Resolution (issue #20 option 1 — `{{`/`}}` escaping):**
- **New pure `render_template(template, &[(key, val)]) -> String`** — single left-to-right pass that substitutes `{key}` AND unescapes `{{`→`{` / `}}`→`}`. This is **not** a `str::replace` add-on: `{{name}}` contains the substring `{name}`, which `replace` would wrongly interpolate. The three templated renderers (`render_base`, `knowledge_intro`, `break_suggestion_intro`) now route through it. Behaviour-equivalent to the old replace-chains for every shipping pack — the `language_guidance` bands are validated placeholder-free (empty allowlist), so there is no nested `{age}` for the old sequential chain to re-substitute.
- **`validate_placeholders`** skips `{{` (escaped literal); a single `{Beispiel}` is still a placeholder attempt and is rejected.
- **Verbatim fields** (everything except the three `*_template` fields) are unescaped at load via `unescape_braces(s) = render_template(s, &[])`, so `{{`/`}}` are literal everywhere under one teachable rule.
- No embedded pack contains doubled braces (verified by grep), so `en`/`de`/`hi` are unaffected.

## Patterns to reuse, not reinvent

New from this session:
- **Brace-escape a placeholder validator like Rust `format!`: `{{`/`}}` → literal, single `{ident}` stays a (validated) placeholder.** The non-obvious trap: you cannot bolt escaping onto a `str::replace("{key}", val)` chain, because `{{key}}` *contains* the substring `{key}` and replace will interpolate it. You need a **single left-to-right pass** that handles escape and substitution together (substituted text is never re-scanned). Verbatim (non-templated) fields get unescaped once at load via the same renderer with an empty var list, so the "doubled braces are literal" rule holds everywhere with no exceptions.
- **When a "future-proofing" code comment describes behaviour the validator already forbids, the future-proofing is dead and you can simplify safely.** `render_base`'s old "substitute language_guidance first in case a band contains `{age}`" ordering was moot — bands are validated with an empty allowlist, so a band *cannot* contain `{age}`. Single-pass rendering is therefore provably equivalent. Verify the validator's allowlist before trusting an ordering comment.
- **Build test TOML bodies with `.replace()` on sentinels (`__BASE__`, `__SUMMARY__`), not `format!`, when the field content under test contains braces** — `format!` treats `{{` as its own escape and corrupts the very syntax you're testing.

Carried forward (prior handoffs): split an over-500-line module into a directory module (`foo/mod.rs` + `foo/tests.rs`) via `git mv` so blame stays put; model a multi-file asset bundle as N single-file `kind`s to reuse single-file download infra; locale-independent asset cache when one model serves all locales; a pure mapping helper for a config gate keeps a Tauri-command decision unit-testable; `--features speech` (feature-gated) clippy is invisible to `--workspace` clippy — run it explicitly; decouple orthogonal backend axes by injecting one into the other's builder (Option A); feature-gated constructor with a rebuild hint; GUI owns mirror config enums when the engine crate is an optional dep; one-shot config migration must be applied at the View layer too; clap `required_if_eq` is blind to default values (ArgGroup + runtime validator); free-form GUI text → parse in pure Rust; a hidden form field must still be sent by `gather()` if its DTO field is mandatory; stream-spanning text transforms need a stateful filter; `generate()` aggregates `generate_stream()`; user-facing strings route through `render_inference_error`; run cargo from `src/` so the 1.88 pin is honored.
