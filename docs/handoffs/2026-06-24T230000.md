## Patterns to reuse, not reinvent

- **CDP over the Android WebView needs `suppress_origin=True`** — the webview devtools rejects the `http://localhost:9222` Origin with 403 otherwise. Helper at scratch `cdp.py` (`python3 cdp.py eval "<js>"`): forwards the `webview_devtools_remote` socket, finds the page target, runs `Runtime.evaluate` with `awaitPromise`+`returnByValue`. Wrap Tauri invokes as `invoke('cmd').then(r=>({status:'ok',r})).catch(e=>({status:'err',e}))` to capture rejections as values.
- **recognizer.log is the on-device truth** (logcat is dead on this ROM): `adb -s 912607710061 shell run-as org.theprimer.gui cat /data/data/org.theprimer.gui/files/recognizer.log`. **Can't truncate via run-as** (ROM denies the redirect even with an absolute path) — use a device-clock baseline (`adb shell date +%s%3N`) and `awk -F' ' '$1 > BASELINE'`.
- **Wedge signature:** a healthy *armed* recognizer fires `onError: 7` every ~5 s even with no speech; any inter-event gap >> 5 s while voice is active (and no committed turn → not a SPEAK gap) is a wedge. SPEAK gaps (50–97 s) are normal — they're the NPU generating + speaking a reply with the mic intentionally disarmed.
- **Force-stop for silence:** `adb shell am force-stop org.theprimer.gui` kills the voice loop + recognizer (stopping voice mode via CDP needs the webview up).
- **Trait-seam + MockBridge** for any new device capability (host-test the pure decision; JNI impl cross-compile-only). **`dyn Trait` method calls need NO `use` of the trait.** **The `android` module is gated behind `--features android-native`** — plain `cargo test -p primer-speech` skips it.
- **APK build:** `cd src/crates/primer-gui && NDK_HOME=/opt/homebrew/share/android-ndk ~/.cargo/bin/cargo-tauri android build --apk --debug --target aarch64 -- --no-default-features --features android-native,qnn` (~4–6 min); APK at `gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk`.
