## What we shipped this session

**Tripwire diagnostic for `KB_BM25_ONLY_MIN_SCORE` floor.** Closes issue #44.

The 0.5 BM25-score floor is a no-op for recall on the current 90-passage corpus, but kept as defensive insurance against future corpus expansion that might dilute term frequencies. Without an automated probe, a regression caused by the floor would be invisible. New `#[ignore]`'d diagnostic at `src/crates/primer-kb-load/tests/bm25_floor_tripwire.rs` probes the actual top-K BM25 score distribution across the 87-query benchmark and asserts:
- `min over all top-K scores > KB_BM25_ONLY_MIN_SCORE`
- `median over all top-K scores > KB_BM25_ONLY_MIN_SCORE × 2.0` (early-warning margin)

**Single commit:**

- `6429d41` — `test(retrieval): add tripwire diagnostic for KB_BM25_ONLY_MIN_SCORE floor`

**Concrete deliverables:**

- New file `src/crates/primer-kb-load/tests/bm25_floor_tripwire.rs` — `#[ignore]`'d tripwire diagnostic + 7 helper unit tests (always-built; cover `min_score`, `median_score`, `sorted_ascending`, `pass_or_fail`).
- **Probe results today:** 435 top-K scores across the 87 queries. min = 0.92 (1.84x floor), median = 4.24 (8.5x floor), min top-1 = 3.35. Both `min > floor` and `median > floor × 2.0` PASS comfortably.
- **TDD-validated:** temporarily tightened margin to 10x → reproduced expected actionable panic with the exact "rerun the sweep / lower the floor / tighten the margin" guidance; reverted to 2.0; PASS again.
- **Doc-comment correction:** the `KB_BM25_ONLY_MIN_SCORE` doc comment in `primer-core/src/consts.rs` previously claimed "every BM25 score in this corpus comfortably exceeds 1.5". Tightened to: every *correct* top-K hit exceeds 1.5; sub-1.5 scores exist as 5th-place noise on marginal queries but don't contribute to recall.
- **Cross-references** added in `CLAUDE.md`, `README.md`, `ROADMAP.md` so future contributors know the tripwire exists and when to run it.
- `cargo fmt --all -- --check` clean. `cargo clippy --workspace --all-targets` clean. Full workspace test count: **605 pass / 0 fail / 2 ignored**.

## Patterns to reuse, not reinvent

(All inherited from prior sessions and confirmed standing.)

- **Pure functions in `primer-core`** for algorithmic cores — tested by injection of `now`, no async, no I/O.
- **`#[cfg(test)]` test seams with `pub` accessor methods** — zero production cost.
- **Locale-keyed templates in TOML packs** with `{placeholder}` substitution at trait method level.
- **Carrier structs with `disabled()` no-op constructors** for parameter bundles that not all callers need to configure.
- **Lookup-table seeding for new closed-enum variants** — no schema migration needed.
- **Constants in `consts.rs` submodules.** No magic numbers anywhere.
- **TDD discipline.** Tests first; watch them fail; implement to green.
- **File-size hygiene.** Keep modules under 500 lines.
- **Network-injection test seam** for any data-ingest pipeline (`http_client` parameter, substitute `FakeHttpClient` in tests).
- **Subagent-driven development workflow** for plan execution.
- **Defensive sanity tests at the data layer.** When test data has multiple consumers, put cheap sanity tests in the shared `tests/common/mod.rs` that validate invariants. They run inside every test binary that imports the module.
- **Always pin `Default` impls of public structs to `consts::*`, with a drift-prevention test.**

**New pattern from this session (re-applied, not invented):**

- **Pure helpers + their own unit tests, even in `#[ignore]`'d integration tests.** The tripwire diagnostic's algorithmic helpers (`min_score`, `median_score`) carry their own `#[cfg(test)] mod helper_tests`. Default CI runs the helpers (cheap, no KB needed) so a future bug in the median computation surfaces immediately. The `#[ignore]`'d wrapper test exercises the helpers against real data. This is the same pattern PRs #43 and #48 used; making it explicit here so future tripwires follow it.
- **Use `min_score: f64::NEG_INFINITY` when probing scores.** When a test wants to observe the actual BM25 score distribution including scores that the production floor would filter, set `min_score = f64::NEG_INFINITY` rather than 0.0 — otherwise the floor masks the very thing you're trying to measure.
