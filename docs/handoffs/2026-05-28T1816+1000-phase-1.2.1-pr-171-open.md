### Implementation topology

- **New crate `src/crates/primer-qnn-sys/`** — three files:
  - `Cargo.toml` (`libloading 0.8` + `thiserror.workspace` as deps; `tokio` macros as dev-dep; `publish = false`).
  - `src/bindings.rs` — hand-rolled `extern "C"` declarations for the six Genie entry points, plus opaque handle types (`GenieDialogConfig`, `GenieDialog`), the `Genie_Status_t` / `Genie_Dialog_SentenceCode_t` aliases, the `GenieDialog_TokenCallback_t` C-ABI signature, and string constants for symbol lookup (`SYM_*`, all NUL-terminated). Module-level doc comment explains the hand-rolled vs bindgen decision and points at the Phase 1.2 risk register that approves it.
  - `src/lib.rs` — `GenieLibrary` struct with the six resolved function pointers + a private `Library` field for lifetime ownership; `GenieLibrary::open(qairt_lib_dir)` is `target_os = "android"`-gated, returning the populated wrapper on success or `GenieLibraryError::PlatformUnsupported` on every other target without touching the filesystem; `GenieLibraryError` enum has three structured variants (`PlatformUnsupported`, `DlopenFailed`, `SymbolMissing`) so the safe wrapper can route on the variant rather than substring-matching messages at the i18n boundary.
- **Workspace plumbing** — `src/Cargo.toml` `[workspace] members` extended with `crates/primer-qnn-sys`.

### TDD record

Four unit tests in the new crate cover the contracts at this layer:

1. `bindings::tests::handle_aliases_are_pointer_sized` — pins that `GenieDialogConfig_Handle_t` and `GenieDialog_Handle_t` are both `*mut c_void`-sized. If a future refactor accidentally aliases them to a struct or to a non-pointer type, this catches the FFI ABI mismatch at compile-test time, not at first call on the device.
2. `bindings::tests::symbol_strings_are_nul_terminated` — `libloading::Library::get::<T>(name)` requires a NUL-terminated byte slice; mis-terminated symbol strings would silently fail to resolve on the device with a misleading error. Pinning the invariant so a typo can't slip past review.
3. `tests::host_open_returns_platform_unsupported` — on macOS/Linux, `GenieLibrary::open` must reject without any filesystem probe. The directory passed does not exist; if the function were attempting to dlopen on a host, we would see a `DlopenFailed`. Asserting `PlatformUnsupported` pins both the short-circuit AND the typed-error contract that the safe wrapper will pattern-match against. The test is cfg-aware so it does the right thing if it ever runs on Termux (Android host with no QAIRT installed: dlopen fails → `DlopenFailed` is the expected variant there).
4. `tests::platform_unsupported_message_mentions_android_and_current_os` — pins the `Display` rendering of the platform-unsupported variant so a future refactor doesn't silently break diagnostic continuity for developers debugging bug reports.

The hand-rolled bindings carry the design rationale and Genie-API references in module-level doc comments so a future contributor swapping the file for `bindgen` output has the prior-art in front of them.

## Patterns to reuse, not reinvent

New from this session:

- **"Verify the cross-compile locally before encoding it in CI as a drift-guard step" is the corollary of PR #167's "verify locally first" rule applied to a new workspace crate.** Before adding `cargo build --target aarch64-linux-android -p primer-qnn-sys` to CI, we ran it on this Mac (no NDK needed because the dep graph is pure-Rust) AND we ran `cargo check --target aarch64-linux-android` AND we ran `cargo clippy --target aarch64-linux-android -- -D warnings`. All three were green before the CI line was added. If any had failed, the right action would have been "fix the crate first, encode the CI line second" — not "add the CI line and let it tell us what's broken."
- **"Document the swap path at the load-bearing line"** — when hand-rolling FFI declarations instead of using `bindgen`, the bindings file's module-level doc comment cites the Phase 1.2 risk register that approves the fallback AND describes how to swap back. A future contributor with the SDK installed can flip to `bindgen` without re-reading any external spec — the rationale and the swap path are co-located with the code. Same shape as the inline `option-(a)-vs-(b)` block at `state_machine.rs:847` and the PR #169 branch-protection comment in `.github/workflows/ci.yml`.
- **"Structured error variants, not stringly-typed `Other(String)`, when the error will cross an i18n boundary."** `GenieLibraryError` has three variants (`PlatformUnsupported`, `DlopenFailed`, `SymbolMissing`) because the safe wrapper in `primer-inference` will pattern-match on the variant to decide what message to render through `primer_core::i18n::render_inference_error`. A single `Other(String)`-style variant would force the i18n layer to substring-match on developer-facing English, which is exactly the bug class the i18n boundary exists to prevent.

Carried forward (from prior sessions; see prior handoff trail for two-layer drain, capacity-1-channel-as-sync-lever, pure-helper extraction, `#[path]` + `#[cfg(test)] mod` two-load pattern for build-script TDD, gate-narrowing, RFC-2229 whole-struct capture, trait-directive refusal documentation patterns, bundle-struct + free-function shape for shared closure builders, one-file-one-cargo-feature-gate as a quick file-size win, public-re-export protecting call sites from internal renames, generic-single-slot-cache + mechanical-caller-wiring as a TDD shape, `Option<T>` taken in finalize-AND-Drop for consume-on-success-AND-on-panic, put-back-before-propagate for caches keyed by expensive construction, "verify locally first, then encode in CI" for feature-combo drift-guards, "`cargo clippy -- -D warnings` over `cargo check` for drift-guards when the rotted-lint bug class exists", "verify the safety assumption before encoding a CI exclusion" for docs-only path filters, and "document the branch-protection coupling at the point of the workaround").
