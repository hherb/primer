### The proven split recipe (reuse it — worked cleanly again this session)
1. Baseline: `cargo test -p <crate>` green BEFORE touching anything; record the pass count.
2. Read the whole file; map natural responsibility boundaries.
3. Convert `foo.rs` → `foo/mod.rs` + sibling submodules; `mod.rs` does `pub use submodule::{…}` for the external API so `crate::foo::Bar` paths are unchanged. (`git rm foo.rs`, then write the new files. NB: when `foo/` already exists as a dir holding `tests.rs`, you're just adding `mod.rs` + siblings alongside it.)
4. **The `use super::*` private-import leak (handled cleanly this time):** a flat module's private `use std::X;` / private helper fns are visible to its `#[cfg(test)] mod tests` via `use super::*`, but a `pub use` re-export façade does NOT carry private `use` aliases, and `pub use submodule::*` does NOT re-export a submodule's *private* items. Two fixes, applied together:
   - Bump any **cross-module helper fn/const** (used by sibling submodules AND/OR tests) to `pub(crate)` in its submodule. This is invisible to the external-pub-surface diff (`pub(crate)` ≠ `pub `).
   - Add the **`use` aliases + submodule-internal items the façade can't carry** explicitly to `tests.rs` (`use std::sync::Arc;`, `use super::render::{…}`, etc.). Keep in `mod.rs` the primer_core `use` aliases the *public trait* needs so `super::*` still carries them (`Locale`, `PedagogicalIntent`, …).
5. Fix any moved relative paths: `super::sibling::…` that moved a level deeper, and — the trap this session — **`include_str!("../…")` becomes `include_str!("../../…")`** when the file drops into a subdir.
6. Verify behaviour-preserving: diff the `pub (struct|enum|fn|const|async fn|trait)` surface old-vs-new (must be empty), re-run the crate suite (count must match baseline), full-workspace `cargo test`, clippy `-D warnings`, fmt `--check`.

## Patterns to reuse, not reinvent

- **The production-split recipe** (6 steps above) — step 4's two-part `use super::*` fix and step 5's `include_str!` depth bump are the two non-obvious traps; everything else is mechanical.
- **Behaviour-preserving proof for a refactor = pub-surface diff + matching test count + clippy + fmt.** The pub-surface diff (`grep -oE 'pub (struct|enum|fn|const|async fn|trait) [A-Za-z_]+' | sort -u` old vs new, must be empty) is the cheap high-signal check that no external symbol got dropped or renamed. Include `async fn` (Tauri commands) and `trait`.
- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift. Cross-check documented *counts* against a fresh grep.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** (Wrong 4×.)
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths.
- **`pgrep -f 'cargo test --workspace'` matches your own waiter loop's command line** (a false "RUNNING"). Trust the background task's exit-code notification + the `tee`'d `/tmp/*.log` result count over `pgrep` for the workspace test. Poll a *specific PID* (`kill -0 <pid>`) when you know it, not the command string.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock and each other's compile jobs (this session paid ~2× for it). Run cargo passes one at a time. (`cargo fmt` does NOT take the target lock, so it can overlap.)
- **A grep `-c` final pipe stage returns exit 1 on zero matches** — benign when the grep is the last stage and the real work passed; check the `Finished` / `test result: ok` line, not the pipeline exit code.
- **Branch each PR off `main`, not off the previous branch.**
