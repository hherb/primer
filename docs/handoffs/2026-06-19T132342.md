## Open decisions / risks

- **The recreate-per-arm fix is unverified on device.** It is the prior brief's planned fix and compiles, but the actual wedge behaviour can only be reproduced on the RedMagic (the emulator has no working on-device recognizer). Until #2 passes, the wedge remains an open risk in practice.
- **~500 ms per-arm create cost.** Recreating the recognizer every arm adds latency before each LISTEN. On the RedMagic this should be acceptable (create cost is dwarfed by the rest of the loop), but watch for a perceptible "dead air" after the Primer stops speaking; if it bites, fall back to a stuck-detection timeout (recreate only when a recognizer goes silent) instead of unconditional recreate. The brief-level decision was "recreate every arm for reliability"; revisit only if latency complaints arise.
- **`en-US` vs installed `en-AU`.** Unchanged from the prior brief: the Kotlin arms with whatever installed same-language variant the device has and triggers a background download otherwise. A fresh device with no installed English would need the (network-gated) download to complete before offline STT works.
- **`recognizer.log` debug sink** (`<filesDir>/recognizer.log`) + `dbg()` calls remain device-diagnostic; keep them (cheap, only window in on a logcat-dead ROM) or gate behind a debug flag for release.
- **Branch not pushed.** Everything this session is local on `android-recognizer-recreate-per-arm`. Push + PR before building on it.
- **Device left as the prior session left it** (`backend.kind=qnn` in `gui-config.json`, QNN APK `--features android-native,qnn` ~193 MB installed). This session touched no device state.

- **Kotlin in `PrimerSpeech.kt` has no host test path — it is device-verified only.** Verify it *compiles* on the host with `./gradlew app:compileUniversalDebugKotlin --offline` from `src/crates/primer-gui/gen/android/` — the dry-run (`--dry-run`) confirms this task does NOT trigger a Rust cross-compile, so it is a fast (~10 s) Kotlin-only check. Use it as the host gate for any `PrimerSpeech.kt` change.
- **The android module mirrors `primer-inference::qnn`:** pure host-tested logic (`should_rearm`, `arm_action`, `AndroidDerivedVad`, `process_event`) + a `MockBridge` test path; the real JNI/Kotlin is device-verified, no host test.
