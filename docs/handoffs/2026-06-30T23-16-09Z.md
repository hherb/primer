### The proven split recipe (reuse it)
1. Baseline: `cargo test -p <crate>` green BEFORE touching anything; record the pass count.
2. Read the whole file; map natural responsibility boundaries.
3. Convert `foo.rs` → `foo/mod.rs` + sibling submodules; `mod.rs` does `pub use submodule::*;` for every submodule so external `crate::foo::Bar` paths are unchanged. (`git rm foo.rs` then write the new files.)
4. **Watch the `use super::*` leak:** a flat module's private `use std::X;` is visible to its `#[cfg(test)] mod tests` via `use super::*`, but a `pub use` re-export façade does NOT carry private `use` aliases — so the test module loses those imports and fails to compile (`E0433 unresolved module`). Fix: add the explicit `use std::X;` the tests need directly to `tests.rs`. (This bit us once this session — `use std::fs;`.)
5. Verify behaviour-preserving: diff the `pub (struct|enum|fn|const)` surface old-vs-new (must be empty), re-run the crate suite (count must match baseline), clippy `-D warnings`, fmt `--check`.

## Patterns to reuse, not reinvent

- **The production-split recipe** (5 steps above) — the `use super::*` private-import leak in step 4 is the one non-obvious trap; everything else is mechanical.
- **Behaviour-preserving proof for a refactor = pub-surface diff + matching test count + clippy + fmt.** The pub-surface diff (`grep -oE 'pub (struct|enum|fn|const) [A-Za-z_]+' | sort` old vs new, must be empty) is the cheap high-signal check that no symbol got dropped or renamed.
- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift. Cross-check documented *counts* against a fresh grep.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.**
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time, `tee` each, grep the captured file. (`cargo fmt` does NOT take the target lock, so it can overlap.)
- **A grep `-c` final pipe stage returns exit 1 on zero matches** — a backgrounded `… | tee log | grep -c` "failing exit 1" is benign when the grep is the last stage and the real work (test/clippy) passed; check the `Finished` / `test result: ok` line, not the pipeline exit code.
- **Branch each PR off `main`, not off the previous branch.**
