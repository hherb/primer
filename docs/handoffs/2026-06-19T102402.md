### 1. ⭐ Push the branch + open the PR, confirm CI green, merge
- **Acceptance:** `git push -u origin android-voice-loop-plan2-tasks7-8`, open a PR, confirm the CI gates (`cargo test (default features)`, `cargo check (non-default features)`, `cargo build (aarch64-linux-android)`, the `primer-gui --features android-native` aarch64 guard, fmt/clippy) are green, merge to `main`.

### 2. Recognizer-reuse robustness — recreate-per-arm (the #1 device follow-up)
- **Problem:** Android's on-device `SpeechRecognizer` degrades/wedges after sustained `startListening` reuse and on a mid-session connectivity (airplane) toggle — it reports `onReadyForSpeech` but then produces no `onPartialResults`/`onResults` and no timeout (`onError: 5`/ERROR_CLIENT recurs). A voice-loop restart (fresh `createOnDeviceSpeechRecognizer`) always recovers it.
- **Fix:** in `PrimerSpeech.kt`, destroy + recreate the `SpeechRecognizer` on each arm (or on a stuck-detection timeout), rather than reusing one instance. Trade ~500ms create cost per arm for reliability. Re-cache the resolved `effectiveTag` so `checkRecognitionSupport` isn't re-run each time.
- **Acceptance:** 10 consecutive voice turns on-device with no wedge; a mid-session airplane toggle does not require a manual restart.

### 3. No-barge-in end-to-end validation (Task 10 Step 4, unblocked by #2)
- **Acceptance:** while the Primer is speaking, the child talking over it does NOT interrupt the Primer and is NOT transcribed as the next turn. The SPEAK-pause code (`arm_action` + `AndroidTts.speaking`) is already in place; re-verify once the recognizer is reliable.

### 4. Leading-word clipping (mitigated, confirm after #2)
- **Acceptance:** the first 1–2 words of a follow-up prompt are not clipped. Already mitigated (immediate re-arm on NO_MATCH, recognizer paused during SPEAK); recreate-per-arm should further help.

## Open decisions / risks

- **Recognizer-reuse wedging is the main open risk for the voice POC.** It's a device/OS-level `SpeechRecognizer` reliability issue, not a flaw in the Primer's voice plumbing (which is proven). Recreate-per-arm (#2 above) is the planned fix; until then the loop needs a restart after extended use / connectivity changes.
- **`en-US` vs installed `en-AU`.** The Kotlin now arms with whatever installed same-language variant the device has; `en-US` was downloaded in-session via `triggerModelDownload`. A fresh device with no installed English would need the (network-gated) download to complete before offline STT works — note this for non-AU/non-pre-provisioned devices.
- **The `recognizer.log` debug sink** (`<filesDir>/recognizer.log`) and the `dbg()` calls in `PrimerSpeech.kt` are device-diagnostic; keep them (cheap, the only window in on a logcat-dead ROM) or gate behind a debug flag if they bother a release build.
- **Branch not pushed.** Everything this session is local on `android-voice-loop-plan2-tasks7-8`. Push + PR before building on it.
- **Device left with `backend.kind=qnn`** in `gui-config.json` (the original); the stub backend was used transiently for the offline test and restored to qnn. The QNN APK (`--features android-native,qnn`, ~193 MB) is the one currently installed.

## Patterns to reuse, not reinvent

- **The android module mirrors `primer-inference::qnn`:** pure host-tested logic (`should_rearm`, `arm_action`, `AndroidDerivedVad`, `process_event`) + a `MockBridge` test path; the real JNI/Kotlin is device-verified, no host test.
- **`run_loop` is untouched by Android** — it takes `on_committed_audio` (no-op), `wait_for_speaker_drain: None`, `is_speaking: None`. Do not modify the shared state machine for Android; coordinate via the `AndroidTts.speaking` flag instead.
- **Device-debug recipe (logcat is dead on this ROM):** symbol check `unzip APK lib/arm64-v8a/*.so` + `llvm-nm -D`; **recognizer behaviour** via `run-as org.theprimer.gui cat files/recognizer.log`; **invoke Tauri commands** over WebView CDP — `adb forward tcp:9222 localabstract:webview_devtools_remote_<pid>`, then a `websocket.create_connection(ws_url, suppress_origin=True)` → `Runtime.evaluate("window.__TAURI__.core.invoke('<cmd>')", awaitPromise=True)`. Helper scripts at `/tmp/cdp_invoke.py` (invoke a command) and `/tmp/cdp_dom.py` (read DOM). `withGlobalTauri` is on.
- **Flip the on-device backend** by editing `gui-config.json` (`backend.kind`): `cat local.json | adb -s 912607710061 shell run-as org.theprimer.gui tee .primer/gui-config.json` (relative path = app data dir under run-as; `cp` is unavailable, `tee` works).
