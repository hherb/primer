### The proven split recipe (reuse it — worked cleanly twice more this session)
1. Baseline: `cargo test -p <crate>` green BEFORE touching anything; record the pass count.
2. Read the whole file; map natural responsibility boundaries.
3. Convert `foo.rs` → `foo/mod.rs` + sibling submodules; `mod.rs` does `pub use submodule::*;` for every submodule so external `crate::foo::Bar` paths are unchanged. (`git rm foo.rs`, then write the new files.)
4. **The `use super::*` private-import leak (bit us on config.rs; handled cleanly this time):** a flat module's private `use std::X;` / private helper fns are visible to its `#[cfg(test)] mod tests` via `use super::*`, but a `pub use` re-export façade does NOT carry private `use` aliases, and `pub use submodule::*` does NOT re-export a submodule's *private* items. Two fixes, applied together:
   - Bump any **private helper fn/const the tests reference** to `pub(crate)` in its submodule (so the `pub use` glob re-exports it). This is invisible to the external-pub-surface diff.
   - Add the **`use` aliases the façade can't carry** (`Arc`, `Utc`, `Uuid`, `AppState`, `std::fs`, …) explicitly to `tests.rs`.
5. Fix any `super::sibling::…` paths that moved a level deeper (e.g. `super::voice::…` → `crate::commands::voice::…`).
6. Verify behaviour-preserving: diff the `pub (struct|enum|fn|const|async fn)` surface old-vs-new (must be empty), re-run the crate suite (count must match baseline), clippy `-D warnings`, fmt `--check`.

## Patterns to reuse, not reinvent

- **The production-split recipe** (6 steps above) — step 4's two-part `use super::*` fix is the one non-obvious trap; everything else is mechanical. It worked cleanly on both splits this session.
- **Behaviour-preserving proof for a refactor = pub-surface diff + matching test count + clippy + fmt.** The pub-surface diff (`grep -oE 'pub (struct|enum|fn|const|async fn) [A-Za-z_]+' | sort -u` old vs new, must be empty) is the cheap high-signal check that no external symbol got dropped or renamed. Include `async fn` — the Tauri commands are `pub async fn`.
- **Doc-drift hunt recipe:** after a feature PR merges, grep the new public symbol across README.md / ROADMAP.md / CLAUDE.md. A symbol present in `src/` but absent from the user-facing docs is the drift. Cross-check documented *counts* against a fresh grep.
- **Re-run the full sweep + inline detector every session; never trust a "backlog exhausted" claim.** (Wrong 4×.)
- **Use ABSOLUTE paths for shell tools** — the Bash cwd resets between calls. Prefix with `cd /Users/hherb/src/primer/src &&` or use full `crates/...` paths.
- **Background-task output capture was flaky this session** — the `<task-id>.output` files sometimes read empty even after the task completed exit 0. Trust the `tee`'d log file (`/tmp/*.log`) and `pgrep` for process liveness over the task-output file. Poll `pgrep -f 'cargo test'` for liveness; grep the tee log for results.
- **Avoid concurrent `cargo` runs** — two overlapping invocations collide on the target lock. Run cargo passes one at a time. (`cargo fmt` does NOT take the target lock, so it can overlap.)
- **A grep `-c` final pipe stage returns exit 1 on zero matches** — benign when the grep is the last stage and the real work passed; check the `Finished` / `test result: ok` line, not the pipeline exit code.
- **Branch each PR off `main`, not off the previous branch.**
