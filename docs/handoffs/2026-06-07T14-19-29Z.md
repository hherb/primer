## Patterns to reuse, not reinvent

New from this session:
- **Ship a deferred extension point as "mechanism now, magic number later" by making the threshold config-driven and OFF by default.** The spec deferred latency routing only to avoid committing an unvalidated TTFT budget; the resolution was an `Option<u64>` budget (CLI flag + GUI field, default `None` = inert). The full mechanism + tests ship; the one genuinely-gated value (the threshold) stays owner-supplied. Use this shape whenever "the design is clear but one constant needs real-world calibration."
- **When a decorator needs per-turn state the bare trait can't carry, own it INSIDE the decorator, not upstream.** The DM holds `&dyn InferenceBackend` and can't see which leg a `RouterBackend` served, so a DM-measured "primary TTFT" would be polluted by cloud turns. The fix: the `RouterBackend` times the primary leg's stream itself (a `TtftTimingStream` wrapper + an `Arc<Mutex<Option<f64>>>` EMA), so attribution is correct by construction and NO upstream (DM/`GenerationParams`) change is needed. Prefer decorator-owned state over threading a signal the upstream can't measure correctly.
- **A nudge term must sit BELOW the decision threshold, and that relationship deserves a drift-tripwire test.** `W_LATENCY < ROUTE_SECONDARY_THRESHOLD` is what makes latency a nudge (tips borderline turns) rather than a circuit-breaker (escalates everything, freezing the self-healing signal). The `router_consts_are_sane` test now asserts it so a future tune can't silently cross the line — same discipline as the exhaustive-match-as-tripwire pattern.
- **Subagent BLOCKED on a planted design inconsistency is the system working.** The Task 3 implementer refused to make the escalation test pass because `Encouragement (0.0) + W_LATENCY (0.30) = 0.30 < threshold (0.5)` — a real flaw in the plan's const sizing AND test design, not a bug to paper over. It surfaced the nudge-vs-circuit-breaker fork to the owner. When an implementer reports BLOCKED with numbers, read them — the plan may be wrong.
- **Split a Rust file that crosses 500 lines into `foo/{mod,tests}.rs`, not an external `tests/` file** — `pub(crate)` test seams (e.g. `with_ttft_ema_for_test`) are visible to a `#[cfg(test)] mod tests;` child module but NOT to an integration test in `tests/` (which is an external crate). `git mv foo.rs foo/mod.rs` preserves history.

Carried forward (prior handoffs): a new inference-routing/resilience behavior is a decorator over `Arc<dyn InferenceBackend>`, built by `build_main_backend`, `name()` → primary, fall over ONLY pre-stream (never mid-stream); rich per-turn signals reach a decorator via an `Option<T>` field on `GenerationParams` every other backend ignores; keep routing/scoring POLICY pure and in `primer-core`, MECHANISM in `primer-inference`; an exhaustive `match` over a closed enum (no wildcard) is a compile-time tripwire; mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (the `Update` DTO has no `#[serde(default)]`, so every new field is MANDATORY in every IPC test payload + `settings.js::gather()`); `RouterMode` serializes PascalCase (`"LocalOnly"`) but its `FromStr`/`name()`/CLI form is kebab (`"local-only"`); adding a non-`Option` field to a shared struct breaks EVERY literal (grep `StructName {`) — `cargo build --tests` may be needed to surface test-only literals; run cargo from `src/` with `+1.88`; docs-only PRs are CI-path-ignored (#168) → BLOCKED → admin-merge/direct-push.
