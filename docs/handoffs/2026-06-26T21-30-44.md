## Patterns to reuse, not reinvent

- **Always run the full sweep before claiming a backlog is empty.** `find crates -name '*.rs' -not -path '*/vendor/*' -not -name 'tests.rs' | xargs wc -l | awk '$1>500'` then `grep -c '#[cfg(test)]'` each. Do this first.
- **Test-module extraction is the lowest-risk, zero-churn oversized-file cut.** Move `#[cfg(test)] mod tests { тАж }` into a sibling file; replace the inline block with `#[cfg(test)]\nmod tests;`. The test module is a *descendant*, so `use super::*` reaches all parent privates (incl. cfg(test) helpers) with no visibility edits. Thirteen PRs in a row (#268тАУ#280) used exactly this. Prefer it over moving production code down (which costs `pub(super)`/`pub(crate)` churn).
- **SINGLE trailing module = naive `sed` works. MULTIPLE sibling test modules = `super`-resolution trap.** Two-or-more sibling `#[cfg(test)] mod X { use super::*; }` blocks, once moved into a child `parent/tests.rs`, each see their `super` re-pointed to `parent::tests` (the file) instead of `parent`. A leading `use super::*;` in the child *may* glob-forward the parent's items to the inner modules (glob-of-glob), but **VERIFY by compiling** тАФ if it fails, change each inner `use super::*;` to `use crate::<parent>::*;`. This is the cloud.rs (2) / wiring.rs (7) situation.
- **The test module name may NOT be `tests`.** A prior session's was `v4_tests`; cloud.rs's second is `classify_tests`. Match the declaration (`mod <name>;`) and child filename to the actual name тАФ grep the `mod тАж` line, don't assume `tests`.
- **Mind the path form: crate root vs. non-root module.** A `lib.rs`/`main.rs` (crate root) submodule resolves to `src/<name>.rs`. A non-root module `foo.rs`'s submodule resolves to `src/foo/<name>.rs`. This session: `voice/assets.rs` тЖТ `src/voice/assets/tests.rs` (new `assets/` dir).
- **Check `#[cfg(feature тАж)]` gating of the MODULE, not just inside the test span.** Here the entire `assets` module is gated on `--features speech` (in `voice/mod.rs`), so a default `cargo test -p primer-gui` never compiles the moved tests тАФ the speech run is the load-bearing regression guard. Same was true for cpal_io (`--features cpal`).
- **Use ABSOLUTE paths or a single confirmed `cd` for `sed`/`head`/`mkdir`** тАФ the Bash cwd persists across calls. Always `git status --short` after.
- **Verify the parent diff is clean:** `git diff -- parent.rs | grep '^+' | grep -v '^+++'` тАФ the only addition should be `mod <name>;`.
- **`cargo fmt` leaves one blank line above the first relocated line that `--check` rejects.** Delete it: `sed -i '' '1{/^$/d;}'`. One-line fix.
- **For a refactor, the test COUNT is the regression guard.** Capture a pre-move baseline (speech 253 here) and confirm the identical count post-move. No new tests authored, none needed.
