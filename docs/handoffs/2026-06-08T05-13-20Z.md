## Patterns to reuse, not reinvent

New from this session:
- **When a file crosses 500 lines *only because of its test module*, the fix is `foo/{mod,tests}.rs` ŌĆö move the tests out, don't fragment the production code.** `backends_common.rs` (554 lines) read well as production helpers; its own TODO note diagnosed the cause correctly (the `#[cfg(test)] mod tests` block). `git mv foo.rs foo/mod.rs` preserves history; replace the inline `mod tests { ŌĆ” }` with `#[cfg(test)] mod tests;`; the moved tests keep `use super::*` (now resolving to the parent `mod.rs`) and only their intra-doc links need `super::` prefixes. Test paths are unchanged because the module path (`ŌĆ”::backends_common::tests::*`) is identical. Same pattern as last session's `router.rs` split ŌĆö now applied twice, so it's the house style for "over-500 due to tests."
- **A pure-relocation refactor's TDD safety net is "the existing tests pass identically before and after."** Establish the baseline by running the exact tests first (here: the 3 `make_on_audio` tests under `--features voice-loop`), do the move, run again, diff the pass/fail. No new test is needed because behaviour didn't change ŌĆö but the before/after run IS the verification, and skipping the baseline would leave you unable to distinguish "refactor was clean" from "tests were already broken."
- **Verify a feature-gated refactor against *every* feature-combo that consumes the moved code, not just one.** `backends_common` is consumed by the whisper builder (`--features voice-loop`) AND the macOS-native builder (`--features voice-loop,macos-native`); clippy on only the first would miss a visibility break in the second. Enumerate the consumers (grep the sibling modules' `use super::backends_common`) and clippy each gate.

Carried forward (prior handoffs): a new inference-routing/resilience behavior is a decorator over `Arc<dyn InferenceBackend>`, built by `build_main_backend`, `name()` ŌåÆ primary, fall over ONLY pre-stream (never mid-stream); rich per-turn signals reach a decorator via an `Option<T>` field on `GenerationParams` every other backend ignores; keep routing/scoring POLICY pure and in `primer-core`, MECHANISM in `primer-inference`; when a decorator needs per-turn state the bare trait can't carry, own it INSIDE the decorator (the `RouterBackend`'s `TtftTimingStream`), not upstream; a nudge term must sit BELOW the decision threshold and that relationship deserves a drift-tripwire test; an exhaustive `match` over a closed enum (no wildcard) is a compile-time tripwire; mirroring a CLI feature into the GUI is a 3-struct (`Config`/`View`/`Update`) + wiring + frontend pattern (the `Update` DTO has no `#[serde(default)]`, so every new field is MANDATORY in every IPC test payload + `settings.js::gather()`); `RouterMode` serializes PascalCase but its `FromStr`/CLI form is kebab; run cargo from `src/` with `+1.88`; docs-only PRs are CI-path-ignored (#168) ŌåÆ BLOCKED ŌåÆ admin-merge/direct-push.
